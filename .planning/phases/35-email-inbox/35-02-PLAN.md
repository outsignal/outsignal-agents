**Thread list API — `src/app/api/portal/inbox/email/threads/route.ts`:**

Create a GET handler that:
1. Calls `getPortalSession()` to get `workspaceSlug`
2. Queries `prisma.reply.findMany({ where: { workspaceSlug }, orderBy: { receivedAt: 'desc' }, take: 200 })`
3. Groups replies into threads using `emailBisonParentId ?? emailBisonReplyId` as the thread key (a Map<number, Reply[]>)
4. For each thread group, builds a summary object:
   ```typescript
   {
     threadId: number,          // The thread root ID
     leadEmail: string,         // From the first inbound reply
     leadName: string | null,   // senderName of the first inbound reply
     subject: string | null,    // Subject from latest message
     lastSnippet: string,       // First ~80 chars of latest message bodyText
     lastMessageAt: string,     // ISO date of most recent message
     messageCount: number,      // Total messages in thread
     interested: boolean,       // True if ANY reply in thread has interested=true
     replyStatus: 'awaiting_reply' | 'replied' | 'new',
     hasAiSuggestion: boolean,  // True if any reply has aiSuggestedReply != null
   }
   ```
5. Derives `replyStatus`:
   - Sort thread messages by receivedAt desc
   - If latest message has `direction === 'outbound'` -> `'replied'`
   - Else if latest message has `notifiedAt === null` -> `'new'`
   - Else -> `'awaiting_reply'`
6. Sorts thread summaries by `lastMessageAt` desc (most recent first)
7. Returns `NextResponse.json({ threads })`

Optional query param `?cursor=ISO_DATE` for pagination: if provided, add `where: { receivedAt: { lt: new Date(cursor) } }` to the query. Response includes `nextCursor` (the `lastMessageAt` of the last thread) if there are exactly 200 replies (indicating more may exist).

**Thread detail API — `src/app/api/portal/inbox/email/threads/[threadId]/route.ts`:**

Create a GET handler that:
1. Calls `getPortalSession()` for workspace auth
2. Parses `threadId` from the route param (parseInt)
3. Queries all replies in the thread:
   ```typescript
   prisma.reply.findMany({
     where: {
       workspaceSlug,
       OR: [
         { emailBisonReplyId: threadId },
         { emailBisonParentId: threadId },
       ],
     },
     orderBy: { receivedAt: 'asc' },
   })
   ```
4. Builds the outbound context message: from the first inbound reply's `outboundSubject` and `outboundBody`, create a synthetic message object:
   ```typescript
   {
     id: 'outbound-context',
     direction: 'outbound',
     subject: firstInbound.outboundSubject,
     bodyText: firstInbound.outboundBody,
     htmlBody: null,
     senderEmail: firstInbound.senderEmail, // This is actually the lead's email; for outbound context, use workspace sender
     receivedAt: null, // No exact date for original outbound
     isOutboundContext: true,
   }
   ```
   Prepend this as the first message if `outboundSubject` or `outboundBody` exists.
5. Maps each Reply to a message object:
   ```typescript
   {
     id: reply.id,
     direction: reply.direction,
     senderEmail: reply.senderEmail,
     senderName: reply.senderName,
     subject: reply.subject,
     bodyText: reply.bodyText,
     htmlBody: reply.htmlBody,
     receivedAt: reply.receivedAt.toISOString(),
     intent: reply.intent,
     sentiment: reply.sentiment,
     interested: reply.interested,
     aiSuggestedReply: reply.aiSuggestedReply,
     ebSenderEmailId: reply.ebSenderEmailId,
     emailBisonReplyId: reply.emailBisonReplyId,
     isOutboundContext: false,
   }
   ```
6. Returns `NextResponse.json({ messages, threadMeta: { leadEmail, leadName, subject, interested } })`

Both routes: wrap in try/catch, return 401 on session error, 500 on unexpected error.

  <action>
Create a POST handler at `src/app/api/portal/inbox/email/reply/route.ts`:

1. Call `getPortalSession()` for `workspaceSlug`
2. Parse request body: `{ replyId: string, message: string }` — where `replyId` is the Prisma Reply.id (cuid) of the inbound reply being responded to, and `message` is the plain text reply body
3. Validate: `message` must be non-empty string (trim and check). Return 400 if empty.
4. Look up the Reply record:
   ```typescript
   const replyRecord = await prisma.reply.findFirst({
     where: { id: replyId, workspaceSlug },
     select: {
       emailBisonReplyId: true,
       ebSenderEmailId: true,
       leadEmail: true,
       emailBisonParentId: true,
     },
   });
   ```
5. Validate: `replyRecord` exists + has `emailBisonReplyId` and `ebSenderEmailId`. Return 422 with `{ error: 'Cannot send: missing sender data for this reply' }` if not.
6. Get the workspace's EmailBison token:
   ```typescript
   const workspace = await prisma.workspace.findUnique({
     where: { slug: workspaceSlug },
     select: { emailBisonToken: true },
   });
   ```
   Return 422 if no `emailBisonToken`.
7. Instantiate `EmailBisonClient` with the token and call:
   ```typescript
   const result = await ebClient.sendReply(replyRecord.emailBisonReplyId!, {
     message: message.trim(),
     sender_email_id: replyRecord.ebSenderEmailId!,
     reply_all: true,  // Per Phase 33 spike: simplest approach, sends to all original recipients
   });
   ```
8. If `!result.data.success`: return `NextResponse.json({ error: result.data.message }, { status: 422 })`
9. On success, persist the sent message as a new Reply record:
   ```typescript
   const sentMsg = result.data.reply!;
   await prisma.reply.create({
     data: {
       workspaceSlug,
       senderEmail: sentMsg.primary_to_email_address ?? replyRecord.leadEmail ?? 'unknown',
       senderName: null,
       subject: sentMsg.subject,
       bodyText: sentMsg.text_body ?? message.trim(),
       htmlBody: sentMsg.html_body ?? null,
       receivedAt: new Date(sentMsg.date_received ?? sentMsg.created_at),
       emailBisonReplyId: sentMsg.id,
       emailBisonParentId: sentMsg.parent_id ?? replyRecord.emailBisonReplyId,
       ebSenderEmailId: sentMsg.sender_email_id,
       leadEmail: replyRecord.leadEmail,
       source: 'portal_send',
       direction: 'outbound',
       interested: false,
     },
   });
   ```
10. Return `NextResponse.json({ success: true })`
11. Wrap the entire handler in try/catch. If `EmailBisonError` (instanceof check), return its statusCode and message. Otherwise return 500.

Import `EmailBisonClient` from `@/lib/emailbison/client` and `EmailBisonError` from `@/lib/emailbison/types`.
