  <files>src/lib/agents/cli-spawn.ts</files>
  <action>
Create `src/lib/agents/cli-spawn.ts` with the following implementation:

1. Export an async function `cliSpawn<T>(scriptName: string, args?: string[]): Promise<T>` that:
   - Resolves scriptPath via `join(process.env.PROJECT_ROOT ?? process.cwd(), 'dist', 'cli', scriptName)`
   - Spawns `node [scriptPath, ...args]` with `child_process.spawn`
   - Passes `stdio: ['ignore', 'pipe', 'pipe']` to prevent stdin hanging (Pitfall 3 from research)
   - Passes `env: { ...process.env }` to inherit all env vars (DATABASE_URL, etc.)
   - Uses `AbortController` with `signal` option for 300s timeout
   - Buffers stdout chunks as `Buffer[]`, concatenates at close, converts to string
   - Buffers stderr chunks for diagnostics
   - On `close` event: parses stdout as JSON, checks `envelope.ok`, resolves with `envelope.data` or rejects with `new Error(envelope.error)`
   - On JSON parse failure: rejects with error including exit code and first 500 chars of stderr
   - On `error` event with AbortController aborted: rejects with `CLI script timed out after 300s: {scriptName}`
   - On other `error` events: rejects with the error directly
   - Always calls `clearTimeout(timer)` in both close and error handlers

2. Export the `CLI_TIMEOUT_MS = 300_000` constant (useful for tests).

3. Export a type `CliResult<T> = T` for type safety at call sites.

Use the exact patterns from the research doc Pattern 1. Do NOT use `execFile` (spawn is preferred for streaming). Do NOT pass secrets as CLI args.

  <name>Task 3: Update Vercel build command to compile CLI scripts</name>
  <files>package.json</files>
  <action>
Update the `"build"` script in `package.json` from:
```json
"build": "prisma generate && next build"
```
to:
```json
"build": "prisma generate && npm run build:cli && next build"
```

This inserts `npm run build:cli` (which runs `tsup --config tsup.cli.config.ts`) between `prisma generate` and `next build`. The order matters:
- `prisma generate` must run first (CLI scripts import Prisma client)
- `build:cli` compiles `scripts/cli/*.ts` into `dist/cli/*.js` via tsup
- `next build` runs last (Next.js build)

This is required because `dist/cli/` is in `.gitignore` -- the compiled scripts do not exist in the git repo and must be built during every Vercel deployment. Without this, every `cliSpawn()` call on Vercel would throw ENOENT.

Do NOT modify `build:cli` itself or any other scripts. Only change the `"build"` line.
  </action>
