<objective>
Refactor the global portal activity API route to use adapter.getActions() for campaign-scoped activity, while retaining direct queries for non-campaign LinkedIn data (messages, connection accepts) as a documented partial migration.

Purpose: PORT-03 — The global activity route (`/api/portal/activity/route.ts`) currently runs 4 parallel Prisma queries: LinkedInAction, Reply, LinkedInMessage, and LinkedInConnection. The research analysis shows that adapter.getActions() covers LinkedInAction and Reply (email). LinkedIn messages and connections are non-campaign entities outside the adapter scope. This plan migrates the campaign-scoped data to adapters and documents the remaining direct queries as intentional.

Output: Global activity API calls adapter.getActions() per channel for each active campaign, merges with non-campaign LinkedIn sources (messages, connections), and returns a unified sorted feed.
