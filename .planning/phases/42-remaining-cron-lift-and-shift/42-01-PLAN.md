Create `trigger/bounce-monitor.ts` — a medium-complexity lift-and-shift from `src/app/api/cron/bounce-monitor/route.ts`. This task replicates the full route logic (not just a single function call).

**Imports:**
```typescript
import { schedules } from "@trigger.dev/sdk";
import { PrismaClient } from "@prisma/client";
import { runBounceMonitor, replaceSender } from "@/lib/domain-health/bounce-monitor";
import { notifySenderHealthTransition, sendSenderHealthDigestEmail, notifyBounceRateTrend } from "@/lib/domain-health/bounce-notifications";
import type { SenderHealthDigestItem } from "@/lib/domain-health/bounce-notifications";
import { detectBounceRateTrend, shouldAlertOnTrend } from "@/lib/domain-health/trend-detection";
import { runReplyTrendMonitor, notifyReplyTrendDecline } from "@/lib/domain-health/reply-trend";
```

**PrismaClient** at module scope: `const prisma = new PrismaClient();`

**Task config:**
- `id: "bounce-monitor"`, `cron: "0 */4 * * *"` (every 4 hours)
- `maxDuration: 300`
- `retry: { maxAttempts: 2, factor: 2, minTimeoutInMs: 5_000, maxTimeoutInMs: 60_000 }`
- No queue — bounce-monitor does NOT call Anthropic. Do NOT apply anthropicQueue.

**`run()` implementation — replicate the full route handler logic:**

1. Call `runBounceMonitor()` to get transitions.

2. Process transitions — for each transition:
   - If `transition.to === "critical"`, attempt `replaceSender()` to find replacement (wrap in try/catch, non-blocking).
   - Call `notifySenderHealthTransition()` with `skipEmail: true` (Slack fires immediately).
   - Collect `SenderHealthDigestItem` for batched email.
   - For warning/critical transitions: dedup-check + create deliverability insight in prisma — copy the exact insight creation logic from the route file (lines 93-145).

3. Bounce rate trend detection — copy from route (lines 157-199):
   - Query active senders from prisma
   - For each sender: `detectBounceRateTrend(email)` → `shouldAlertOnTrend()` → `notifyBounceRateTrend()`
   - Count trend alerts

4. Reply trend monitoring — copy from route (lines 201-227):
   - Call `runReplyTrendMonitor()`
   - For each declining workspace: `notifyReplyTrendDecline()`
   - Count reply trend alerts

5. Send combined digest email if `digestItems.length > 0`: `sendSenderHealthDigestEmail(digestItems)`

6. Log summary and return: `{ evaluated, transitioned, skipped, trendAlerts, replyTrendAlerts }`

**Key difference from the three thin wrappers:** This task needs prisma for insight creation and sender queries. The bounce-monitor route file has ~260 lines of orchestration logic that must be lifted wholesale.

**Export:** `export const bounceMonitorTask = schedules.task({ ... });`
