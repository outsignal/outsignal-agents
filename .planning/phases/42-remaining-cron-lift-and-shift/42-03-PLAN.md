PrismaClient at module scope (needed for person/company lookups in the enrichment callback).

Task config:
- `id: "enrichment-job-processor"`, `cron: "0 6 * * *"` (daily 6am UTC — matches current vercel.json schedule)
- `maxDuration: 300`, `retry: { maxAttempts: 2, factor: 2, minTimeoutInMs: 5_000, maxTimeoutInMs: 60_000 }`
- No queue

`run()` — replicate the handleProcess logic from the enrichment route:
1. Create fresh circuit breaker: `const breaker = createCircuitBreaker()`
2. Call `processNextChunk(async (entityId, job) => { ... })` with the enrichment callback that:
   - If `job.entityType === "person"`: look up person from prisma, call `enrichEmail()`
   - If `job.entityType === "company"`: look up company from prisma, call `enrichCompany()`
3. If result is null: return `{ message: "no pending jobs" }`
4. Otherwise return the result object

**Important:** The current Vercel cron only processes ONE chunk per invocation (designed for repeated calls). In Trigger.dev we have 300s — consider calling `processNextChunk` in a loop until it returns null (all jobs done). This is a natural improvement: instead of processing one chunk per day, process ALL pending chunks. Wrap in a while loop:

```typescript
const results = [];
let chunk = await processNextChunk(enrichFn);
while (chunk) {
  results.push(chunk);
  if (chunk.done) break;
  chunk = await processNextChunk(enrichFn);
}
return { chunksProcessed: results.length, results };
```
