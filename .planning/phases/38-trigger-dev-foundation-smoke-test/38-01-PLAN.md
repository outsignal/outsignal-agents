<task type="auto">
  <name>Task 2: Update Prisma schema binaryTargets and define shared queues</name>
  <files>prisma/schema.prisma, trigger/queues.ts</files>
  <action>
**Prisma schema update:**

In `prisma/schema.prisma`, update the `generator client` block to add `binaryTargets`. The current block is:
```prisma
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["postgresqlExtensions"]
}
```

Change it to:
```prisma
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["postgresqlExtensions"]
  binaryTargets   = ["native", "debian-openssl-3.0.x"]
}
```

`"native"` keeps local dev working. `"debian-openssl-3.0.x"` is required for Trigger.dev Cloud's runtime environment.

After editing, run `npx prisma generate` to regenerate the client with the new binary target.

**Shared queues:**

Create `/trigger/queues.ts` with two pre-declared queues using the `queue()` function from `@trigger.dev/sdk`:

```typescript
import { queue } from "@trigger.dev/sdk";

export const anthropicQueue = queue({
  name: "anthropic-queue",
  concurrencyLimit: 3,
});

export const emailBisonQueue = queue({
  name: "emailbison-queue",
  concurrencyLimit: 3,
});
```

These queues MUST be pre-declared with `queue()` â€” in Trigger.dev v4, inline concurrency limits set at trigger time are silently ignored. All future AI tasks will import `anthropicQueue` and all EmailBison tasks will import `emailBisonQueue`.
