  <action>
Create `trigger/smoke-test.ts` with a `task()` definition that tests all five services sequentially, reporting per-service pass/fail results.

The task structure:
```typescript
import { task } from "@trigger.dev/sdk";
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

export const smokeTest = task({
  id: "smoke-test",
  // No queue — this is a diagnostic tool, not a production workload
  // No schedule — on-demand only
  run: async (payload: Record<string, never>) => {
    const results: Record<string, { ok: boolean; ms: number; detail?: string; error?: string }> = {};
    // ... test each service
    return results;
  },
});
```

**Service tests (sequential, each wrapped in try/catch with timing):**

1. **Prisma/Neon** — `prisma.person.findFirst()` — proves DB connectivity and correct binary target
   - ok: true if a Person record is returned
   - detail: person email (first 3 chars + "***")

2. **Anthropic** — `generateText({ model: anthropic("claude-haiku-4-5"), prompt: "Reply with exactly: OK" })`
   - Import `anthropic` from `@ai-sdk/anthropic` and `generateText` from `ai`
   - ok: true if response text includes "OK"
   - detail: response text (first 50 chars)

3. **Slack** — `new WebClient(process.env.SLACK_BOT_TOKEN).auth.test()`
   - Import `WebClient` from `@slack/web-api`
   - ok: true if response.ok is true
   - detail: bot user ID from response

4. **EmailBison** — `fetch("https://app.outsignal.ai/api/workspaces", { headers: { Authorization: "Bearer " + process.env.EMAILBISON_API_KEY } })`
   - Simple GET request to verify the API key works
   - ok: true if response status is 200
   - detail: response status code

5. **Resend** — `new Resend(process.env.RESEND_API_KEY).apiKeys.list()`
   - Import `Resend` from `resend`
   - ok: true if the call returns without error
   - detail: number of API keys returned

Each test should measure duration in ms (Date.now() before/after) and include it in the result.

After all tests, compute a summary:
```typescript
const allPassed = Object.values(results).every(r => r.ok);
const summary = { allPassed, services: Object.keys(results).length, passed: Object.values(results).filter(r => r.ok).length };
return { summary, results };
```

IMPORTANT:
- Use `new PrismaClient()` directly — no globalForPrisma pattern (tasks run in isolated processes)
- Read env vars at runtime from `process.env` — Trigger.dev injects them from the dashboard
- Do NOT import from `src/lib/` — task files should be self-contained to avoid Next.js bundling issues
- Keep the task permanent — it's a diagnostic tool for debugging connectivity after deploys
