<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Trigger.dev project and connect integrations</name>
  <action>
User must complete these steps in the Trigger.dev dashboard (https://cloud.trigger.dev):

**Step 1: Create the project**
1. Log in with jonathan@outsignal.ai
2. Create a new project named `outsignal-agents`
3. Note the project ref (e.g., `proj_xxxx`) — needed for trigger.config.ts

**Step 2: Connect GitHub**
1. Go to Project Settings -> Git
2. Connect the GitHub repository for outsignal-agents
3. This is a prerequisite for the Vercel integration

**Step 3: Connect Vercel integration**
1. Go to Project Settings -> Connect Vercel
2. Authorize Trigger.dev app on Vercel
3. Select the Vercel project (`cold-outbound-dashboard` or `outsignal-agents`)
4. Review env vars — select ALL to sync
5. Enable atomic deployments for production

**Step 4: Override DATABASE_URL**
1. In Trigger.dev Dashboard -> Project Settings -> Environment Variables
2. Find DATABASE_URL (synced from Vercel)
3. Edit it to append `?connection_limit=1` to the pooled Neon URL
4. This prevents connection exhaustion when multiple tasks run concurrently

**Step 5: Check Neon IP allowlisting**
1. Go to Neon Console -> Project Settings -> IP Allow
2. If IP allowlisting is enabled, add Trigger.dev Cloud IP ranges
3. If not enabled (likely), skip — document as "not enabled"

**Step 6: Get local dev secret key**
1. In Trigger.dev Dashboard -> Project Settings -> API Keys
2. Copy the DEV environment secret key (starts with `tr_dev_...`)
3. Provide it to Claude for .env.local
