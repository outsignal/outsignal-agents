```typescript
export interface BodyElements {
  hasCtaType: boolean;
  ctaSubtype: "book_a_call" | "reply_to_email" | "visit_link" | "download_resource" | null;
  hasProblemStatement: boolean;
  hasValueProposition: boolean;
  hasCaseStudy: boolean;
  hasSocialProof: boolean;
  hasPersonalization: boolean;
}
```

Export `EMPTY_ELEMENTS` constant (all false/null).

Create `classifyBodyElements(emailBody: string): Promise<BodyElements>` that:
- Returns EMPTY_ELEMENTS if body is empty or < 20 chars
- Uses `generateText` from `ai` with `anthropic("claude-haiku-4-5-20250315")` (same model as strategy-detect.ts)
- Uses the classification prompt from RESEARCH.md (6 element definitions + CTA subtypes)
- Parses JSON response with try/catch, validates and coerces each field
- Falls back to EMPTY_ELEMENTS on parse failure (log error, don't throw)

Create `classifyWorkspaceBodyElements(workspaceSlug: string): Promise<{ classified: number; skipped: number; errors: string[] }>` that:
- Loads all Campaign records for the workspace that have non-null emailSequence
- For each campaign, parses emailSequence JSON to get step array
- For each step with a `body` field:
  - Check if CachedMetrics already has a `body_elements` row for this campaign+step with matching content hash
  - If already classified with same content hash, skip
  - Otherwise call `classifyBodyElements(step.body)`
  - Upsert into CachedMetrics: `metricType: "body_elements"`, `metricKey: "{campaignId}:step:{position}"`, `date: ""` (elements don't change daily)
  - Store `data: JSON.stringify({ ...elements, bodyHash: md5-like-hash })` using a simple string hash (use built-in crypto createHash('md5').update(body).digest('hex'))
- Returns counts of classified, skipped, and any errors

The content hash pattern: compute a hash of `step.body`, store it in the CachedMetrics data JSON alongside the element flags. On next run, compare hash to detect if email body was edited (and re-classify if so). Use Node.js `crypto.createHash('md5').update(body).digest('hex')`.
