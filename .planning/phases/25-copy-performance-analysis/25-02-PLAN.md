Create `GET /api/analytics/copy/subject-lines` following the pattern from `src/app/api/analytics/campaigns/route.ts`.

Query params: `workspace` (optional), `vertical` (optional), `view` ("global" | "per-campaign", default "global"), `sort` ("openRate" | "replyRate" | "sends", default "replyRate"), `order` ("asc" | "desc", default "desc").

Implementation:
1. Call `requireAdminAuth()` (import from `@/lib/auth`)
2. Fetch latest `campaign_snapshot` rows from CachedMetrics (group by metricKey to get latest date per campaign, same pattern as campaigns/route.ts)
3. For each campaign snapshot, load the Campaign record to get `emailSequence`
4. Parse emailSequence JSON, extract each step's `subjectLine` and `subjectVariantB`
5. For workspace filter: filter by CachedMetrics.workspace
6. For vertical filter: join through Campaign.workspaceSlug -> Workspace.vertical
7. Cross-reference with snapshot's stepStats to get reply counts per step. Use campaign-level `emailsSent` as denominator for rates (step-level sent not available). Compute `openRate` from campaign-level `openRate`. Compute `replyRate` per step as `stepReplied / emailsSent * 100`.
8. Minimum 10 sends threshold: exclude campaigns with < 10 emailsSent
9. For "global" view: deduplicate identical subject line text, aggregate metrics (sum sends, weighted-average rates across campaigns using the same subject line)
10. For "per-campaign" view: group by campaign, list each step's subject line
11. Include `subjectVariantB` as separate entries when present (noted as "Variant B" in response)
12. Sort by requested metric, return array

Response shape:
```typescript
{
  subjectLines: Array<{
    text: string;
    campaignCount: number; // how many campaigns use this (global view)
    campaignName: string | null; // null for global, set for per-campaign
    step: number | null; // null for global
    totalSends: number;
    openRate: number;
    replyRate: number;
    isVariantB: boolean;
  }>;
  total: number;
  view: "global" | "per-campaign";
  filters: { workspace: string | null; vertical: string | null };
}
```

**Correlations route** (`GET /api/analytics/copy/correlations`):

Query params: `workspace` (optional), `vertical` (optional).

Implementation:
1. `requireAdminAuth()`
2. Fetch all `body_elements` CachedMetrics rows (optionally filtered by workspace)
3. Fetch latest `campaign_snapshot` rows for each campaign (same pattern as subject lines route)
4. For vertical filter: look up each campaign's workspace, then Workspace.vertical
5. Build a list of `StepWithElements` objects: for each body_elements row, parse the campaignId and step position from metricKey (`{campaignId}:step:{position}`), find the matching campaign snapshot's stepStats entry, combine element flags with step reply metrics
6. For each of the 6 boolean element fields (hasCtaType, hasProblemStatement, hasValueProposition, hasCaseStudy, hasSocialProof, hasPersonalization):
   - Compute global multiplier: weighted average reply rate of steps WITH element / steps WITHOUT element (weighted by emailsSent from campaign snapshot). Use campaign-level emailsSent since step-level sent not available.
   - Compute vertical-specific multiplier: same calculation filtered to the selected vertical (or the campaign's vertical if no filter). Return null if either bucket has 0 steps.
   - Track sample sizes (count of steps with/without)
7. Additionally compute CTA subtype breakdown: for steps where hasCtaType is true, group by ctaSubtype and compute average reply rate per subtype
8. Guard against division by zero: if either with or without count is 0, return `multiplier: null`
9. Minimum confidence: flag correlations where `sampleWith + sampleWithout < 20` as `lowConfidence: true`

Response shape:
```typescript
{
  correlations: Array<{
    element: string; // "cta_type", "problem_statement", etc. (snake_case display names)
    displayName: string; // "CTA", "Problem Statement", etc.
    globalMultiplier: number | null;
    globalSampleWith: number;
    globalSampleWithout: number;
    verticalMultiplier: number | null;
    verticalSampleWith: number;
    verticalSampleWithout: number;
    verticalName: string | null;
    lowConfidence: boolean;
  }>;
  ctaSubtypes: Array<{
    subtype: string;
    avgReplyRate: number;
    sampleSize: number;
  }>;
  totalStepsAnalyzed: number;
  filters: { workspace: string | null; vertical: string | null };
}
```

**Top templates route** (`GET /api/analytics/copy/top-templates`):

Query params: `workspace` (optional), `vertical` (optional), `limit` (default 10).

Implementation:
1. `requireAdminAuth()`
2. Fetch latest `campaign_snapshot` rows + `body_elements` rows from CachedMetrics (same data assembly as correlations)
3. For each campaign step that has both body_elements and snapshot data:
   - Get campaign record to retrieve emailSequence step body text and subject line
   - Compute composite score: `(interestedRate * 0.6) + (replyRate * 0.4)` using campaign-level rates
   - Only include campaigns with >= 10 emailsSent
4. Sort by composite score descending, take top `limit`
5. For vertical filter: filter by workspace's vertical

Response shape:
```typescript
{
  templates: Array<{
    campaignId: string;
    campaignName: string;
    workspaceSlug: string;
    step: number;
    subjectLine: string;
    body: string; // full email body text
    elements: BodyElements; // from body_elements CachedMetrics
    replyRate: number;
    interestedRate: number;
    compositeScore: number;
    totalSends: number;
    copyStrategy: string | null;
  }>;
  total: number;
  filters: { workspace: string | null; vertical: string | null };
}
```
