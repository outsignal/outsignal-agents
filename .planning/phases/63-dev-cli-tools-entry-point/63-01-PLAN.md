  <action>
Create 9 dev-cli scripts following the exact harness pattern from the research. Each script:
- Does NOT need dotenv (git/code tools don't need DB; deploy-status loads .env and .env.local)
- Imports `{ runWithHarness }` from `./_cli-harness`
- Imports `{ execSync }` from `child_process` (or `fs` for read-file)
- Parses args from `process.argv` using simple positional or `--flag value` parsing (no yargs)
- Sets `maxBuffer: 10 * 1024 * 1024` on all execSync calls
- Sets `cwd: process.env.PROJECT_ROOT || process.cwd()` on all execSync calls
- Returns structured JSON matching the fields in the research tool inventory

Script specifications:

**git-status.ts**: No args. Runs `git status --porcelain` and `git branch --show-current`. Returns `{ branch, clean, files: [{status, path}] }`.

**git-diff.ts**: Optional arg: target (default "HEAD"). Runs `git diff {target} --stat` and `git diff {target} --numstat`. Returns `{ target, summary, files: [{file, added, removed}], totalAdded, totalRemoved }`.

**git-log.ts**: Optional `--count N` arg (default 20). Runs `git log --format="%H|%an|%ai|%s" -n {count}`. Returns `{ commits: [{hash, author, date, message}], count }`.

**read-file.ts**: Required `--path <filepath>`. Optional `--start-line N --end-line N`. Uses `fs.readFileSync`. Returns `{ path, content, lineCount, sizeBytes }`. If start/end lines given, returns only that range. Error if file doesn't exist.

**list-files.ts**: Optional `--pattern <glob>` (default `*`), optional `--path <dir>` (default `.`). Uses `find` with exclusions for node_modules, dist, .next, .git, .trigger. Returns `{ pattern, directory, files: string[], count }`. Limit to 500 results.

**search-code.ts**: Required `--pattern <regex>`. Optional `--glob <fileglob>` (e.g. `*.ts`), optional `--limit N` (default 50). Uses `grep -rn` with `--max-count=5` per file. Excludes node_modules, dist, .next, .git, .trigger. Returns `{ pattern, glob, matches: [{file, line, content}], matchCount, truncated }`.

**run-tests.ts**: Optional positional arg: test path. Uses `npx vitest run --reporter=json` (with optional path). Parse JSON output. Returns `{ passed: bool, failed: number, total: number, duration: string, failures: [{test, error}] }`. If JSON parse fails, fall back to text parsing of exit code.

**check-types.ts**: No args. Runs `npx tsc --noEmit`. Returns `{ passed: bool, errorCount, errors: [{file, line, col, code, message}] }`. Parse TS error format: `file(line,col): error TSXXXX: message`. Limit to 50 errors.

**deploy-status.ts**: No args. Loads dotenv (.env + .env.local). Tries `vercel ls --json 2>/dev/null`. If that fails, tries `vercel --version` for availability check. Returns deployment info or `{ available: false, message }`. Wrap in try/catch for graceful degradation.

IMPORTANT: Do NOT import dotenv in git/code/test scripts — only deploy-status.ts needs it. The harness itself does not load dotenv (unlike Nova's harness which does).
