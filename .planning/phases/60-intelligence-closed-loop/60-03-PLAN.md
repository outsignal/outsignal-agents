  <action>
Create `scripts/run-reply-analysis.ts` -- a CLI script that runs the full analysis pipeline.

1. Load dotenv (.env then .env.local)
2. Import analyzeWorkspace, analyzeCrossWorkspace, synthesizeInsights from reply-analysis module
3. Import appendToMemory from agents/memory
4. Get all workspaces with replies: `prisma.reply.groupBy({ by: ['workspaceSlug'], _count: true })`
5. For each workspace with replies:
   a. Run `analyzeWorkspace(slug)` to get per-workspace metrics
   b. Print a summary to console (reply count, top campaign, sentiment breakdown)
6. Run `analyzeCrossWorkspace()` to get cross-workspace metrics
7. For each workspace, run `synthesizeInsights(workspaceData, crossData, slug)`:
   a. Write workspaceInsights to campaigns.md via `appendToMemory(slug, 'campaigns.md', insight)`
   b. Collect globalInsights (deduplicate across workspaces)
8. Write global insights to `.nova/memory/global-insights.md`:
   - Direct file write (not appendToMemory, which only works for per-workspace files)
   - Read existing file content first
   - Check line count (respect 200-line max)
   - Format: `[{ISO timestamp}] [Vertical: {vertical}] -- {insight text}`
   - Append new insights, preserving existing content
9. Also store insights in the Insight DB table (type: 'weekly_analysis') for the sync script and for Vercel/Trigger.dev access:
   ```typescript
   await prisma.insight.create({
     data: {
       workspaceSlug: slug,
       type: 'weekly_analysis',
       content: JSON.stringify({ globalInsights, workspaceInsights }),
     },
   });
   ```
10. Print final report: total insights generated, per-workspace breakdown, files written

Run with: `npx tsx scripts/run-reply-analysis.ts`

Add a `--week-comparison` flag that, when set, loads the previous week's analysis from the Insight table (type='weekly_analysis', createdAt within last 14 days) and includes a comparison section in the synthesized insights (e.g., "Reply rate improved from X to Y").

The script should be idempotent for the same week -- if run multiple times, it appends new insights (appendToMemory deduplication prevents exact duplicates).
