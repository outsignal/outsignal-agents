Create `trigger/weekly-analysis.ts` -- a Trigger.dev scheduled task that runs the reply analysis pipeline every Monday at 09:00 UTC. This fulfills the locked CONTEXT.md decision: "Weekly cron (Monday) via Trigger.dev".

Follow the existing pattern from `trigger/generate-insights.ts`:

```typescript
import { schedules } from "@trigger.dev/sdk/v3";
import { analyzeWorkspace, analyzeCrossWorkspace, synthesizeInsights } from "../src/lib/reply-analysis";
import { prisma } from "../src/lib/prisma";

// Use anthropicQueue if it exists in the codebase, otherwise create inline
// Check how generate-insights.ts imports its queue

export const weeklyAnalysis = schedules.task({
  id: "weekly-analysis",
  cron: "0 9 * * 1", // Monday 09:00 UTC (after generate-insights at 08:10)
  maxDuration: 120, // 2 min
  retry: {
    maxAttempts: 2,
    factor: 2,
    minTimeoutInMs: 5_000,
    maxTimeoutInMs: 30_000,
  },
  run: async () => {
    // 1. Get all workspaces with replies
    const workspaces = await prisma.reply.groupBy({
      by: ["workspaceSlug"],
      _count: true,
    });

    // 2. Gather data for each workspace
    const workspaceResults = new Map();
    for (const ws of workspaces) {
      const data = await analyzeWorkspace(ws.workspaceSlug);
      workspaceResults.set(ws.workspaceSlug, data);
    }

    // 3. Cross-workspace analysis
    const crossData = await analyzeCrossWorkspace();

    // 4. Synthesize insights per workspace and store to DB
    // NOTE: Trigger.dev runs remotely -- cannot write to local memory files.
    // Store to Insight DB table. User runs sync-insights-to-memory.ts locally.
    for (const [slug, wsData] of workspaceResults) {
      const { globalInsights, workspaceInsights } = await synthesizeInsights(wsData, crossData, slug);

      await prisma.insight.create({
        data: {
          workspaceSlug: slug,
          type: "weekly_analysis",
          content: JSON.stringify({ globalInsights, workspaceInsights }),
        },
      });
    }

    return {
      workspacesAnalyzed: workspaces.length,
      timestamp: new Date().toISOString(),
    };
  },
});
```

Key design decisions:
- Runs AFTER generate-insights (08:10) to avoid queue contention
- Stores results in Insight DB table (type: 'weekly_analysis') -- NOT local memory files (Trigger.dev runs remotely)
- User runs `npx tsx scripts/sync-insights-to-memory.ts` locally to pull DB insights into memory files
- Uses the same retry pattern as other Trigger.dev tasks
- maxDuration: 120 seconds (analysis + LLM synthesis for ~6 workspaces)

Check how other trigger/ files import shared modules (e.g. prisma, queue). Use the same import pattern. The import path from trigger/ to src/ is typically `../src/lib/...`.
