Create `src/lib/reply-analysis.ts` with three exported functions. The first two are pure data gathering (Prisma queries only, no LLM calls), and the third is LLM synthesis.

**1. `analyzeWorkspace(slug: string): Promise<WorkspaceAnalysis>`**

Runs Prisma queries for a single workspace:
- Reply count by campaign (groupBy campaignId + campaignName, _count)
- Reply count by sequenceStep (groupBy sequenceStep where not null)
- Sentiment distribution (groupBy sentiment)
- Intent distribution (groupBy intent)
- Objection subtype breakdown (groupBy objectionSubtype where intent='objection')
- Top outbound subjects by positive reply count (groupBy outboundSubject where sentiment='positive', orderBy count desc, take 10)
- Top outbound subjects by total reply count (groupBy outboundSubject where not null, orderBy count desc, take 10)
- Load CachedMetrics for this workspace's campaigns (type='campaign_snapshot') to get reply rates, open rates

Return a typed WorkspaceAnalysis object with all these breakdowns.

**2. `analyzeCrossWorkspace(): Promise<CrossWorkspaceAnalysis>`**

Runs cross-workspace Prisma queries:
- All workspaces with replies (groupBy workspaceSlug, _count, include workspace vertical)
- Strategy performance by workspace vertical (join Campaign.strategy with Reply counts -- requires fetching campaigns with their strategies and correlating)
- Universal step patterns (groupBy sequenceStep across all workspaces)
- Subject line length correlation: for replies with outboundSubject, compute average word count of subjects that got positive vs negative replies
- Per-vertical benchmarks: reply counts + CachedMetrics rates grouped by workspace vertical

Return a typed CrossWorkspaceAnalysis object.

**3. `synthesizeInsights(workspaceData: WorkspaceAnalysis, crossData: CrossWorkspaceAnalysis, slug: string): Promise<{ globalInsights: string[]; workspaceInsights: string[] }>`**

Takes the raw analysis data and produces human-readable insight strings:
- For global insights: format as `[Vertical: {vertical}] -- {pattern with specific numbers}` (the ISO timestamp will be added by appendToGlobalMemory)
- For workspace insights: format as `{workspace}: {copy insight with specific numbers}`
- Use an LLM call (anthropic Opus 4.6 via AI SDK) to synthesize patterns from the data. Pass the raw numbers as a structured prompt, ask for 5-10 specific, data-backed insights per category.
- Prompt should ask for insights about: which strategies work best, which sequence steps generate most replies, subject line patterns, objection patterns, sentiment trends.

Define TypeScript interfaces for WorkspaceAnalysis and CrossWorkspaceAnalysis at the top of the file. Export them.

For the LLM call in synthesizeInsights: use the same `generateText` + `anthropic` pattern used throughout the codebase. Model: the NOVA_MODEL constant from `src/lib/agents/types.ts` (or import it).
