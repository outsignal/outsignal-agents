<objective>
Add the validateAllChecks() aggregator to copy-quality.ts, add a validateCopy tool to the writer, enhance save tools with full validation, and update writer-rules.md with intent-based anti-pattern descriptions and the self-review protocol.

Purpose: The writer currently only checks banned patterns at save time. This plan adds: (1) an aggregator that runs ALL structural checks in one call, (2) a tool the writer can call to validate before saving, (3) defense-in-depth in save tools, and (4) rules updates that teach the LLM WHY patterns are banned and mandate the generate-validate-rewrite loop.

  <name>Validation Aggregator + Writer Validate Tool + Save Tool Enhancement + Rules Update</name>
  <files>src/lib/copy-quality.ts, src/lib/__tests__/copy-quality.test.ts, src/lib/agents/writer.ts, .claude/rules/writer-rules.md</files>
  <behavior>
    **1. validateAllChecks() in copy-quality.ts:**
    - Accepts text, field type, and options (strategy, channel, isFirstStep)
    - Runs checks based on field and channel:
      - body + email: checkWordCount(text, strategy), checkGreeting(text, isFirstStep), checkCTAFormat(text), checkCopyQuality(text) -> convert violations to CheckResult[]
      - body + linkedin: checkWordCount(text, "linkedin"), checkLinkedInSpintax(text), checkGreeting(text, isFirstStep), checkCopyQuality(text) -> convert violations
      - subject/subjectVariantB: checkSubjectLine(text), checkCopyQuality(text) -> convert violations
    - Returns StepValidationResult with field name, all CheckResult violations, and hasHardViolation flag
    - Null results from individual checks are filtered out (no violation)

    **2. validateCopy tool in writer.ts:**
    - Accepts array of steps (position, subject, subjectVariantB, body, channel) and strategy
    - Calls validateAllChecks() for each field of each step
    - Returns structured result per step: { position, violations: StepValidationResult[], clean: boolean }
    - If all clean: { status: "clean", message: "All steps pass quality checks" }
    - If violations: { status: "violations_found", steps: [...], summary: "Step 1 body: word count 85 exceeds 70 (pvp)..." }

    **3. Enhanced save tools:**
    - saveCampaignSequence: before saving, run validateAllChecks() on each step's body, subject, subjectVariantB. Block on any hard violation. Allow soft violations with a warning in the response.
    - saveDraft: same enhancement — run validateAllChecks() on body, subject, subjectVariantB fields.
    - Both return quality_violation status with full CheckResult details on hard violations.

    **4. writer-rules.md updates:**
    - Group banned phrases by intent category in Shared Quality Rules:
      - "Fake-casual engagement bait" (quick question, genuine question, honest question, curious if, curious whether, ring any bells, sound familiar)
      - "Self-introduction filler" (My name is, I wanted to reach out)
      - "Lazy follow-up language" (just following up, following up, touching base, circling back, circle back)
      - "Corporate buzzwords" (synergy, leverage, streamline, game-changer, revolutionary)
      - "Urgency/pressure tactics" (guaranteed, act now, limited time, exclusive offer, no obligation, free)
      - "Over-eager tone" (excited to, I'd love to, we'd love to, pick your brain)
      - "Passive/permission-seeking" (no worries if not, no worries at all, feel free to, at your earliest convenience, as per my last email)
      - "Formatting violations" (em dash, en dash, hyphen separator, double-brace variables, lowercase variables)
    - Add new "Self-Review Protocol" section after FINAL CHECK:
      ```
      ## Self-Review Protocol (MANDATORY)

      Before saving ANY copy (via saveCampaignSequence or saveDraft), you MUST:
      1. Generate the complete sequence
      2. Call validateCopy with ALL steps, the strategy, and channel
      3. If violations found: rewrite the offending steps to fix violations
      4. Call validateCopy again with corrected steps
      5. If still violations: rewrite once more (attempt 2 of 2)
      6. Call validateCopy a final time
      7. If STILL violations after 2 rewrites: save anyway, but add "[REVIEW NEEDED] Violation: {description}" to the notes field of each affected step
      8. Only then call saveCampaignSequence or saveDraft

      NEVER call save tools without first calling validateCopy. The save tools will reject hard violations anyway (defense-in-depth), but pre-validation catches issues earlier and enables rewrites.
      ```
  </behavior>
