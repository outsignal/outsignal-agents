```typescript
// Write orchestrator memory after delegation turns
const delegationCalls = result.steps
  .flatMap((s) => s.toolCalls)
  .filter((tc) => tc.toolName.startsWith("delegateTo"));

if (delegationCalls.length > 0 && workspaceSlug) {
  const actions = delegationCalls
    .map((tc) => tc.toolName.replace("delegateTo", ""))
    .join(", ");
  try {
    await appendToMemory(
      workspaceSlug,
      "learnings.md",
      `Orchestrator session: delegated to ${actions}`,
    );
  } catch {
    // Best-effort — memory write failure should never break chat
  }
}
```

**Key requirements per CONTEXT.md decisions:**
- Only write when tool calls were made (the `delegationCalls.length > 0` guard handles this)
- Skip pure-text responses (handled by the filter — no delegation calls = no write)
- Keep it lightweight — one-line entry with delegation targets listed
- Best-effort — wrapped in try/catch, never throws

Per Pitfall 4 from research: only write when delegation tool calls are present (delegateToResearch, delegateToWriter, etc.) or mutation tools. Skip pure-query turns. The `startsWith("delegateTo")` filter achieves this correctly.
