### Tasks

#### Task 1: Invoice CRUD API routes + Sender settings API

**Type:** auto

**Files:**
- `src/app/api/invoices/route.ts`
- `src/app/api/invoices/[id]/route.ts`
- `src/app/api/invoice-settings/route.ts`

**Action:**

First, install `@react-pdf/renderer`:
```bash
npm install @react-pdf/renderer
```

1. **Create `src/app/api/invoices/route.ts`:**
   - `GET`: List invoices. Query params: `workspaceSlug` (optional), `status` (optional). Calls `listInvoices()` from operations. Returns JSON array of invoices with lineItems.
   - `POST`: Create invoice. Body: `{ workspaceSlug, lineItems: [{description, quantity, unitPricePence}], billingPeriodStart?, billingPeriodEnd?, issueDate? }`. Calls `createInvoice()`. Returns 201 with created invoice.
   - Error handling: try/catch, return appropriate 400/500 with `{ error: message }`.

2. **Create `src/app/api/invoices/[id]/route.ts`:**
   - `GET`: Fetch single invoice by ID. Also accept `?token=xxx` query param for token-based access (portal PDF download). If token provided, use `getInvoiceByToken()`. Otherwise use `getInvoice(id)`. Return 404 if not found.
   - `PATCH`: Update invoice status. Body: `{ status: "sent" | "paid" }`. Calls `updateInvoiceStatus()`. Returns updated invoice. Returns 400 if invalid transition.

3. **Create `src/app/api/invoice-settings/route.ts`:**
   - `GET`: Fetch the first (and only) InvoiceSenderSettings record. Return 200 with data, or 200 with null if none exists yet.
   - `PUT`: Upsert sender settings. Body: `{ senderName, senderAddress?, senderEmail, accountNumber?, sortCode? }`. Use `prisma.invoiceSenderSettings.upsert()` — if an existing record exists, update it; if not, create it. Return 200 with saved record.

No auth guards on any of these routes — consistent with all other admin API routes (15-04 decision pattern).

**Verify:**
```bash
npx tsc --noEmit 2>&1 | grep -E "invoices|invoice-settings" | head -10
```

**Done:** GET/POST /api/invoices works. GET/PATCH /api/invoices/[id] works with status transition validation. GET/PUT /api/invoice-settings manages sender settings.

#### Task 2: PDF generation + Invoice email delivery

**Type:** auto

**Files:**
- `src/lib/invoices/pdf.tsx`
- `src/app/api/invoices/[id]/pdf/route.ts`
- `src/lib/invoices/email.ts`
- `src/app/api/invoices/[id]/send/route.ts`

**Action:**

1. **Create `src/lib/invoices/pdf.tsx`:**
   Use `@react-pdf/renderer` to create a professional invoice PDF matching the Google Doc format from CONTEXT.md specifics:
   - Import `Document, Page, Text, View, StyleSheet` from `@react-pdf/renderer`
   - Page size A4, padding 48pt, Helvetica font, font size 10pt, color #18181b
   - Layout:
     - **Header**: Large "INVOICE" text (24pt, bold)
     - **Two-column header row**: Sender details (name, address, email) on left; Bill-to (company name, address) on right
     - **Invoice metadata row**: Invoice #, Date (DD/MM/YYYY), Due Date, Amount Due — displayed in a horizontal bar
     - **Line items table**: Headers: Items | Description | Quantity | Price | Amount. Rows for each line item. Use `formatGBP` for prices. Alternating row shading optional.
     - **Totals section** (right-aligned): Subtotal, Tax (show rate %), Tax Amount, **Total** (bold)
     - **Notes section** (bottom-left): "NOTES" header, bank details: "Account Number: {accountNumber}", "Sort Code: {sortCode}"
   - Export `InvoicePdfDocument` component accepting `{ invoice: InvoiceWithLineItems }` prop
   - Use `formatGBP` from `@/lib/invoices/format` for all currency display
   - Use `formatInvoiceDate` for date formatting

   **IMPORTANT:** This file is server-only. Never import in a "use client" component. Only import from API routes.

2. **Create `src/app/api/invoices/[id]/pdf/route.ts`:**
   - `GET`: Renders PDF and returns as downloadable file
   - Accept optional `?token=xxx` query param for portal access (use `getInvoiceByToken` if token present, else `getInvoice`)
   - Import `renderToBuffer` from `@react-pdf/renderer`
   - Render `<InvoicePdfDocument invoice={invoice} />`
   - Return `new Response(buffer, { headers: { "Content-Type": "application/pdf", "Content-Disposition": \`attachment; filename="${invoice.invoiceNumber}.pdf"\` } })`
   - Return 404 if invoice not found

3. **Create `src/lib/invoices/email.ts`:**
   - `invoiceEmailHtml(invoice: InvoiceWithLineItems): string` — Returns branded HTML email string following the project's existing email pattern (see `src/lib/resend.ts` onboarding email as reference). Layout:
     - OUTSIGNAL header bar (black bg, #F0FF7A brand text)
     - Body: "Invoice {invoiceNumber}" heading, "Please find your invoice attached for the period {billingPeriodStart} - {billingPeriodEnd}.", amount due, due date
     - CTA button: "View Invoice" linking to `/api/invoices/${invoice.id}/pdf?token=${invoice.viewToken}`
     - Footer: "Outsignal" attribution
   - `sendInvoiceEmail(invoice: InvoiceWithLineItems, recipientEmail: string): Promise<void>`:
     - Import Resend from "resend"
     - Import `renderToBuffer` from `@react-pdf/renderer` and `InvoicePdfDocument` from pdf.tsx
     - Generate PDF buffer
     - Send via `resend.emails.send()` with `attachments: [{ filename: \`${invoice.invoiceNumber}.pdf\`, content: pdfBuffer }]`
     - Use `process.env.RESEND_FROM ?? "Outsignal <notifications@outsignal.ai>"` as from address

4. **Create `src/app/api/invoices/[id]/send/route.ts`:**
   - `POST`: Send invoice email
   - Fetch invoice via `getInvoice(id)`, return 404 if not found
   - Determine recipient: use workspace's `billingClientEmail` (fetch workspace by `invoice.workspaceSlug`). If not set, return 400 with error "No billing email configured for this workspace"
   - Call `sendInvoiceEmail(invoice, recipientEmail)`
   - Update invoice status to "sent" via `updateInvoiceStatus(id, "sent")` — this also sets `sentAt`
   - Return 200 with `{ sent: true, to: recipientEmail }`

**Done:** PDF generation produces a professional A4 invoice. Email delivery sends via Resend with PDF attachment. Send endpoint updates status to "sent" automatically.
