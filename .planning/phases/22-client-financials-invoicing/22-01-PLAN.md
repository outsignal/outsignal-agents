### Tasks

#### Task 1: Prisma Schema — Invoice models + Workspace billing fields

**Type:** auto

**Files:**
- `prisma/schema.prisma`

**Action:**

Add the following models to `prisma/schema.prisma` (at the end, before any closing comments):

1. **InvoiceSenderSettings** model — global sender details for invoices:
   - `id` String @id @default(cuid())
   - `senderName` String — e.g. "Jonathan Jay"
   - `senderAddress` String? — multi-line personal address
   - `senderEmail` String
   - `accountNumber` String? — bank account number
   - `sortCode` String? — e.g. "12-34-56"
   - `updatedAt` DateTime @updatedAt

2. **Invoice** model:
   - `id` String @id @default(cuid())
   - `invoiceNumber` String @unique — e.g. "PS03"
   - `workspaceSlug` String
   - Bill-from snapshot fields: `senderName` String, `senderAddress` String?, `senderEmail` String, `bankDetails` String?
   - Bill-to snapshot fields: `clientCompanyName` String, `clientAddress` String?
   - Date fields: `issueDate` DateTime, `dueDate` DateTime, `billingPeriodStart` DateTime?, `billingPeriodEnd` DateTime?
   - Financial fields (integer pence): `subtotalPence` Int @default(0), `taxRate` Float @default(0), `taxAmountPence` Int @default(0), `totalPence` Int @default(0)
   - `status` String @default("draft") — draft | sent | paid | overdue
   - Delivery tracking: `sentAt` DateTime?, `paidAt` DateTime?
   - Auto-generation: `autoGenerated` Boolean @default(false), `renewalDate` DateTime?
   - Portal access: `viewToken` String? @unique
   - Overdue tracking: `reminderSentAt` DateTime? — prevents duplicate reminder emails
   - Timestamps: `createdAt` DateTime @default(now()), `updatedAt` DateTime @updatedAt
   - Relation: `lineItems InvoiceLineItem[]`
   - Indexes: `@@index([workspaceSlug])`, `@@index([status])`, `@@index([issueDate])`, `@@index([dueDate])`, `@@index([workspaceSlug, status])`

3. **InvoiceLineItem** model:
   - `id` String @id @default(cuid())
   - `invoiceId` String
   - `description` String — e.g. "Cold Outbound Retainer (26/01/2026 - 23/02/2026)"
   - `quantity` Int @default(1)
   - `unitPricePence` Int
   - `amountPence` Int — quantity * unitPricePence
   - Relation: `invoice Invoice @relation(fields: [invoiceId], references: [id], onDelete: Cascade)`
   - Index: `@@index([invoiceId])`

4. **InvoiceSequence** model:
   - `id` String @id @default(cuid())
   - `workspaceSlug` String @unique
   - `lastNumber` Int @default(0)

5. **Workspace billing fields** — add to the existing Workspace model (after the signalWatchlistDomains field, before createdAt):
   - `billingCompanyName` String?
   - `billingAddressLine1` String?
   - `billingAddressLine2` String?
   - `billingCity` String?
   - `billingPostcode` String?
   - `invoicePrefix` String? — e.g. "PS" for Paytier Solutions
   - `invoiceTaxRate` Float @default(0) — e.g. 20 for 20% VAT
   - `billingRenewalDate` DateTime? — client's monthly renewal date
   - `billingRetainerPence` Int? — monthly retainer amount in pence
   - `billingPlatformFeePence` Int? — monthly platform fee in pence
   - `billingPaymentTermsDays` Int @default(5) — due date = issue date + N days
   - `billingDaysBeforeRenewal` Int @default(7) — generate invoice N days before renewal
   - `billingClientEmail` String? — email address to send invoices to (may differ from clientEmails)

After schema changes, run `npx prisma db push` to apply.

**IMPORTANT:** Store ALL monetary amounts as integer pence (e.g. £500.00 = 50000). NEVER use Float for currency amounts.

2. **Create `src/lib/invoices/format.ts`:**
   ```typescript
   export function formatGBP(pence: number): string {
     return new Intl.NumberFormat("en-GB", {
       style: "currency",
       currency: "GBP",
     }).format(pence / 100);
   }

   export function penceToPounds(pence: number): string {
     return (pence / 100).toFixed(2);
   }

   // Format date as DD/MM/YYYY for invoice display
   export function formatInvoiceDate(date: Date): string {
     return date.toLocaleDateString("en-GB", {
       day: "2-digit",
       month: "2-digit",
       year: "numeric",
     });
   }
   ```

3. **Create `src/lib/invoices/numbering.ts`:**
   ```typescript
   import { prisma } from "@/lib/db";

   export async function getNextInvoiceNumber(
     workspaceSlug: string,
     prefix: string
   ): Promise<string> {
     const result = await prisma.$transaction(async (tx) => {
       const seq = await tx.invoiceSequence.upsert({
         where: { workspaceSlug },
         create: { workspaceSlug, lastNumber: 1 },
         update: { lastNumber: { increment: 1 } },
       });
       return seq.lastNumber;
     });
     return `${prefix}${String(result).padStart(2, "0")}`;
   }
   ```

4. **Create `src/lib/invoices/operations.ts`:**
   - `createInvoice(input: CreateInvoiceInput)`:
     - Fetches workspace (for billing fields + invoicePrefix)
     - Fetches InvoiceSenderSettings (first record — there's only one)
     - Generates invoice number via `getNextInvoiceNumber`
     - Snapshots sender details and client billing address into the invoice record
     - Computes subtotalPence (sum of lineItem amountPence), taxAmountPence (Math.round(subtotalPence * taxRate / 100)), totalPence
     - Sets dueDate = issueDate + workspace.billingPaymentTermsDays days
     - Generates a `viewToken` via `crypto.randomUUID()`
     - Creates invoice with nested lineItems in a single prisma.invoice.create
     - Returns the created invoice with lineItems
   - `getInvoice(id: string)`: findUnique with include lineItems
   - `getInvoiceByToken(token: string)`: findUnique where viewToken, include lineItems
   - `listInvoices(filters?: { workspaceSlug?: string; status?: string })`: findMany with include lineItems, orderBy issueDate desc
   - `updateInvoiceStatus(id: string, newStatus: InvoiceStatus)`:
     - Fetches current invoice
     - Validates transition against VALID_TRANSITIONS
     - Updates status + sets sentAt/paidAt timestamps as appropriate
     - If marking as "paid", advances workspace.billingRenewalDate by 1 calendar month
   - `advanceRenewalDate(date: Date): Date`: returns new Date with month incremented by 1, handles month-end edge cases (Jan 31 -> Feb 28)

   **CRITICAL:** All monetary math uses integer pence. No floats for amounts. Tax calculation: `Math.round(subtotalPence * taxRate / 100)`.
