### Tasks

#### Task 1: Portal sidebar + Billing page + Portal invoice API

**Type:** auto

**Files:**
- `src/components/portal/portal-sidebar.tsx`
- `src/components/portal/portal-mobile-menu.tsx`
- `src/app/api/portal/invoices/route.ts`
- `src/app/(portal)/portal/billing/page.tsx`

**Action:**

1. **Update `src/components/portal/portal-sidebar.tsx`:**
   - Add `Receipt` to the lucide-react import
   - Add billing nav item to `navItems` array:
     ```typescript
     { href: "/portal/billing", label: "Billing", icon: Receipt },
     ```
   - Place it after the LinkedIn item in the array

2. **Update `src/components/portal/portal-mobile-menu.tsx`:**
   - Add the same "Billing" nav item to match sidebar. (Read the file first to understand its structure — it likely mirrors the sidebar navItems.)

3. **Create `src/app/api/portal/invoices/route.ts`:**
   - `GET`: Returns invoices for the authenticated portal user's workspace
   - Use `getPortalSession()` to get `workspaceSlug`
   - Query `prisma.invoice.findMany({ where: { workspaceSlug }, include: { lineItems: true }, orderBy: { issueDate: "desc" } })`
   - Filter out draft invoices (only show sent, paid, overdue to clients)
   - Return JSON array
   - If session fails, return 401

4. **Create `src/app/(portal)/portal/billing/page.tsx`:**
   - Server component
   - Use `getPortalSession()` to get workspaceSlug
   - Fetch invoices from `prisma.invoice.findMany` (server component can query directly) — where `workspaceSlug` and status NOT "draft"
   - Layout:
     - Page title: "Billing" (styled consistently with other portal pages)
     - If no invoices: empty state message "No invoices yet."
     - Invoice table with columns:
       - Invoice # — text
       - Date — formatted DD/MM/YYYY
       - Due Date — formatted DD/MM/YYYY
       - Amount — formatGBP(totalPence)
       - Status — badge (same InvoiceStatusBadge component or inline badge)
       - Action — "Download PDF" link pointing to `/api/invoices/${invoice.id}/pdf?token=${invoice.viewToken}` (opens in new tab)
     - Use project's Table components from `@/components/ui/table`
   - Style: Match other portal pages (clean, professional, consistent with portal dashboard)

**IMPORTANT:** PDF download links use the `viewToken` query parameter, NOT admin auth. This allows direct PDF access from the portal without the portal user having admin credentials. The PDF route handler already supports `?token=` access from Plan 02.
