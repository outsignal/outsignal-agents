Create the three Trigger.dev scheduled task files that replace the cron-job.org + Vercel API route pattern for retry-classification, generate-insights, and snapshot-metrics.

**Imports:**
```typescript
import { schedules } from "@trigger.dev/sdk";
import { PrismaClient } from "@prisma/client";
import { classifyReply } from "@/lib/classification/classify-reply";
import { anthropicQueue } from "./queues";
```

**PrismaClient** at module scope: `const prisma = new PrismaClient();`

**Task config:**
- `id: "retry-classification"`
- `cron: "*/30 * * * *"` (every 30 minutes)
- `queue: anthropicQueue`
- `maxDuration: 300` (5 min — enough for all unclassified replies)
- `retry: { maxAttempts: 2, factor: 2, minTimeoutInMs: 1_000, maxTimeoutInMs: 10_000 }`

**`run()` implementation:**

1. Query ALL unclassified replies — `prisma.reply.findMany({ where: { classifiedAt: null }, orderBy: { createdAt: "asc" } })`. NO `take` limit (CRON-01 requirement: no batch size limit).

2. Process each reply sequentially (Anthropic calls are queued anyway):
   ```typescript
   let classified = 0;
   let failed = 0;

   for (const reply of unclassified) {
     try {
       const classification = await classifyReply({
         subject: reply.subject,
         bodyText: reply.bodyText,
         senderName: reply.senderName,
         outboundSubject: reply.outboundSubject,
         outboundBody: reply.outboundBody,
       });

       await prisma.reply.update({
         where: { id: reply.id },
         data: {
           intent: classification.intent,
           sentiment: classification.sentiment,
           objectionSubtype: classification.objectionSubtype,
           classificationSummary: classification.summary,
           classifiedAt: new Date(),
         },
       });

       classified++;
       console.log(`[retry-classification] Classified reply ${reply.id}: intent=${classification.intent}, sentiment=${classification.sentiment}`);
     } catch (err) {
       failed++;
       console.error(`[retry-classification] Failed to classify reply ${reply.id}:`, err);
     }
   }
   ```

3. Return summary: `{ total: unclassified.length, classified, failed }`

**Export:** `export const retryClassification = schedules.task({ ... });`

- `queue: anthropicQueue`

2. Define `processWorkspace(slug: string)` as a local function (replicate exactly from the Vercel route — runs snapshot, strategy backfill, and body element classification with per-step error isolation):
   ```typescript
   async function processWorkspace(slug: string) {
     const errors: string[] = [];

     const { campaignsProcessed, errors: snapshotErrors } = await snapshotWorkspaceCampaigns(slug);
     errors.push(...snapshotErrors);

     let strategiesBackfilled = 0;
     try {
       strategiesBackfilled = await backfillCopyStrategies(slug);
     } catch (err) {
       errors.push(`Strategy backfill failed: ${err instanceof Error ? err.message : String(err)}`);
     }

     let elementsClassified = 0;
     let elementsSkipped = 0;
     try {
       const result = await classifyWorkspaceBodyElements(slug);
       elementsClassified = result.classified;
       elementsSkipped = result.skipped;
       if (result.errors.length > 0) errors.push(...result.errors);
     } catch (err) {
       errors.push(`Body element classification failed: ${err instanceof Error ? err.message : String(err)}`);
     }

     return { workspace: slug, campaignsProcessed, strategiesBackfilled, elementsClassified, elementsSkipped, errors: errors.length > 0 ? errors : undefined };
   }
   ```

3. Fan out with `Promise.all` — process all workspaces in parallel:
   ```typescript
   const results = await Promise.all(
     workspaces.map(async (ws) => {
       try {
         return { ok: true, ...(await processWorkspace(ws.slug)) };
       } catch (err) {
         return { ok: false, workspace: ws.slug, error: err instanceof Error ? err.message : String(err) };
       }
     })
   );
   ```

4. Return summary: `{ workspacesProcessed: results.length, results }`

- `cron: "0 */6 * * *"` (every 6 hours)
- `queue: anthropicQueue`
- `maxDuration: 300` (5 min — all workspaces with AI insight generation)
- `retry: { maxAttempts: 2, factor: 2, minTimeoutInMs: 2_000, maxTimeoutInMs: 30_000 }`

**`run()` implementation:**

2. Define `sendDigestForWorkspace(workspaceSlug: string)` — copy the digest logic from the Vercel route file (`src/app/api/cron/generate-insights/route.ts` lines 12-101). This function:

   - Queries latest campaign snapshots for best/worst campaign
   - Counts active insights for pending actions
   - Counts replies in last 7 days
   - Computes avg reply rate
   - Calls `notifyWeeklyDigest()` with all gathered data
   - Wraps everything in try/catch (digest failure is non-blocking)

3. Fan out with `Promise.all` — process all workspaces in parallel:
   ```typescript
   const results = await Promise.all(
     workspaces.map(async (ws) => {
       try {
         const count = await generateInsights(ws.slug);
         await sendDigestForWorkspace(ws.slug);
         return { workspace: ws.slug, insightsGenerated: count, digestSent: true };
       } catch (err) {
         return { workspace: ws.slug, insightsGenerated: 0, error: err instanceof Error ? err.message : String(err) };
       }
     })
   );
   ```

   const totalInsights = results.reduce((s, r) => s + r.insightsGenerated, 0);

     totalInsightsGenerated: totalInsights,
     digestsSent: workspaces.length - errors.length,
     results,
     errors: errors.length > 0 ? errors : undefined,
