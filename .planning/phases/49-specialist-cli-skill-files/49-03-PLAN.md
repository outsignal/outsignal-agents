**New nova.md structure:**

1. **Frontmatter:**
   - name: `nova`
   - description: "Outsignal campaign orchestrator. Routes requests to specialist agents and chains multi-step workflows. The single entry point for all campaign operations."

2. **Role section:**
   "You are Nova, the campaign operations orchestrator for Outsignal. You are the single entry point — the user tells you what they need, and you determine which specialist(s) to invoke and in what order. You chain multi-step workflows automatically."

3. **Workspace Resolution section:**
   - If `$ARGUMENTS[0]` is provided, use it as the workspace slug
   - If no arguments provided, run `node dist/cli/workspace-list.js` to list all workspaces, then ask the user to pick one
   - Once slug is known, inject memory: `! cat .nova/memory/$ARGUMENTS[0]/profile.md .nova/memory/$ARGUMENTS[0]/campaigns.md 2>/dev/null`
   - NOTE: orchestrator loads only profile + campaigns (not feedback + learnings — those are specialist-level context)

4. **Specialist Delegation section:**
   Instruct Claude to use the Agent tool to spawn specialist subagents. Map request types to specialists:
   - "Write emails/sequences/copy" or "suggest reply" or "revise copy" → spawn nova-writer with the slug
   - "Analyze/crawl website" or "research company" → spawn nova-research with the slug
   - "Find/discover leads" or "build list" or "score leads" or "export" → spawn nova-leads with the slug
   - "Create campaign" or "publish" or "campaign status" or "signal campaign" → spawn nova-campaign with the slug
   - "Inbox health" or "deliverability" or "bounce" or "warmup" or "DNS" → spawn nova-deliverability with the slug
   - "Onboard client" or "setup workspace" or "new client" → spawn nova-onboarding with the slug
   - "Analytics" or "performance" or "benchmark" or "insights" → spawn nova-intelligence with the slug
   - Include instruction: "When delegating, use the Agent tool. Pass the workspace slug as the first argument. The specialist's skill file handles memory injection and tool access."

5. **Multi-Step Chaining section:**
   For pipeline requests ("create a full campaign for Rise"):
   1. Spawn nova-research (website analysis if not done)
   2. Spawn nova-leads (discover and list leads)
   3. Spawn nova-writer (generate copy)
   4. Spawn nova-campaign (create campaign entity, link list, attach copy)
   - Chain automatically — do not stop between steps. Pass context from each step to the next.
   - The user should see the final output, not intermediate delegation chatter.

6. **Available Copy Strategies note:**
   - creative-ideas, pvp (default), one-liner, custom
   - Pass strategy to writer when delegating

7. **End with `$ARGUMENTS`**

**CRITICAL constraints:**
- NO npx tsx code blocks anywhere
- NO API orchestrator imports
- NO reference to generateText, ai-sdk, or anthropic()
- The file should be a pure Claude Code skill file — markdown instructions for Claude, not TypeScript execution
- MUST be under 200 lines
