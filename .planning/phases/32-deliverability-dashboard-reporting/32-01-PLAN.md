  <action>
Create two GET API endpoints with admin auth (requireAdminAuth pattern from existing routes).

**GET /api/deliverability/summary**
- Accept optional `?workspace=slug` query param
- Query `prisma.domainHealth.findMany()` — count healthy vs at-risk (warning/critical) domains
  - If workspace filter: join through Sender table to find domains matching that workspace's senders (extract domain from sender emailAddress)
- Find worst domain: the one with `overallHealth === 'critical'` or highest `blacklistSeverity`, fallback to first `'warning'`
- Query `prisma.sender.groupBy({ by: ['emailBounceStatus'], _count: true, where: { emailAddress: { not: null } } })` — count senders by health status
  - Apply workspace filter if provided
- Query `prisma.emailHealthEvent.findMany({ take: 5, orderBy: { createdAt: 'desc' } })` — recent transitions (quick preview)
  - Apply workspace filter if provided
- Return JSON: `{ domains: { total, healthy, atRisk, worst: { domain, overallHealth } }, senders: { total, healthy, elevated, warning, critical }, recentEvents: [...] }`

**GET /api/deliverability/domains**
- Accept optional `?workspace=slug` query param
- Query `prisma.domainHealth.findMany({ orderBy: { domain: 'asc' } })`
- For each domain, count active senders: `prisma.sender.count({ where: { emailAddress: { endsWith: '@' + domain } } })`
  - Optimize: batch this with a single groupBy query on Sender by domain extracted via raw SQL or do a findMany and count in JS
  - If workspace filter: add `workspaceSlug` to sender count where clause
- Parse JSON fields: `dkimSelectors` and `blacklistHits` (stored as JSON strings — parse with JSON.parse)
- Return JSON array: `[{ domain, spfStatus, dkimStatus, dkimSelectors: string[], dmarcStatus, dmarcPolicy, blacklistHits: string[], blacklistSeverity, overallHealth, lastDnsCheck, lastBlacklistCheck, activeSenderCount }]`

Both endpoints: wrap in try/catch, return 500 with `{ error: message }` on failure.

<task type="auto">
  <name>Task 2: Senders and Events API endpoints</name>
  <files>src/app/api/deliverability/senders/route.ts, src/app/api/deliverability/events/route.ts</files>
  <action>
Create two GET API endpoints with admin auth.

**GET /api/deliverability/senders**
- Accept optional `?workspace=slug` query param
- Query all senders with emailAddress not null: `prisma.sender.findMany({ where: { emailAddress: { not: null }, ...(workspace ? { workspaceSlug: workspace } : {}) }, orderBy: { emailAddress: 'asc' } })`
- For each sender, fetch last 30 days of BounceSnapshot data for sparkline:
  `prisma.bounceSnapshot.findMany({ where: { senderEmail: sender.emailAddress, snapshotDate: { gte: thirtyDaysAgo } }, orderBy: { snapshotDate: 'asc' }, select: { snapshotDate: true, bounceRate: true } })`
  - Optimize: batch query all snapshots with `senderEmail: { in: senderEmails }` then group by senderEmail in JS
- Return JSON array: `[{ id, emailAddress, workspaceSlug, emailBounceStatus, emailBounceStatusAt, warmupDay, warmupStartedAt, currentBounceRate: (latest snapshot bounceRate or null), sparklineData: [{ date: string, bounceRate: number }], consecutiveHealthyChecks }]`
- `currentBounceRate`: take the most recent bounceSnapshot's bounceRate for this sender, or null if no data
- `sparklineData`: map snapshotDate to ISO date string, bounceRate to number (handle null as 0)

**GET /api/deliverability/events**
- Accept optional `?workspace=slug` and `?cursor=eventId` query params
- Default page size: 20
- Query: `prisma.emailHealthEvent.findMany({ take: 21, ...(cursor ? { skip: 1, cursor: { id: cursor } } : {}), orderBy: { createdAt: 'desc' }, where: workspace ? { workspaceSlug: workspace } : {} })`
- Take 21 to detect if more exist: if result.length === 21, pop the last, set `hasMore: true`
- Return JSON: `{ events: [{ id, senderEmail, senderDomain, workspaceSlug, fromStatus, toStatus, reason, bouncePct, detail, createdAt }], hasMore, nextCursor: lastEvent.id }`

Both endpoints: wrap in try/catch, return 500 with `{ error: message }` on failure.
