  1. `prisma.domainHealth.findMany()` — count healthy vs at-risk (warning/critical)

  4. `prisma.sender.findMany({ where: { emailBounceStatus: { in: ['warning', 'critical'] }, emailAddress: { not: null } } })` — current problem senders
  5. Per-workspace bounce trends: for each workspace, get latest BounceSnapshot avg bounceRate vs 7-day-ago avg bounceRate, compute up/down arrow
- Build Slack message blocks:
  - Header: ":chart_with_upwards_trend: Weekly Deliverability Digest"
  - Section: "Domains: X healthy, Y at-risk"
  - If worst domain: "Worst: {domain} ({overallHealth})"
  - Section: "Transitions this week: N"
  - If problem senders: list each sender email + status
  - Section: "Bounce Trends by Workspace:" — list each workspace with avg bounce rate and ↑/↓ arrow vs last week
  - Action button: "View Deliverability Dashboard" linking to https://admin.outsignal.ai/deliverability
- Build email HTML: same content in a styled HTML email (follow existing Outsignal email template pattern from other notifications in this file)
  - Subject: "[Outsignal] Weekly Deliverability Digest"
