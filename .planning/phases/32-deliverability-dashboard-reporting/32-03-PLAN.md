- For each transition where `toStatus === 'warning' || toStatus === 'critical'`:
  - Create a dedup key: `deliverability_${workspaceSlug}_${senderEmail}_${toStatus}`
  - Check dedup: look for an existing active insight with this key pattern (query `prisma.insight.findFirst({ where: { category: 'deliverability', status: 'active', workspaceSlug, observation: { contains: senderEmail } } })`)
    - If an existing dedup insight is found using the pattern from `src/lib/insights/dedup.ts`, skip creation
  - If no dupe: create insight:
    ```
    prisma.insight.create({
      data: {
        category: 'deliverability',
        observation: `Sender ${senderEmail} has reached ${toStatus} status with ${bouncePct}% bounce rate`,
        evidence: JSON.stringify({ senderEmail, fromStatus, toStatus, bouncePct, reason }),
        actionType: toStatus === 'critical' ? 'pause_sender' : 'flag_copy_review',
        actionDescription: toStatus === 'critical'
          ? `Consider pausing ${senderEmail} — bounce rate ${bouncePct}% exceeds critical threshold`
          : `Monitor ${senderEmail} closely — bounce rate ${bouncePct}% is elevated`,
        status: 'active',
        workspaceSlug,
        priority: toStatus === 'critical' ? 1 : 2,
      }
    })
    ```
- Import prisma if not already imported in this file
- Wrap insight creation in try/catch — insight failure should not break the cron
