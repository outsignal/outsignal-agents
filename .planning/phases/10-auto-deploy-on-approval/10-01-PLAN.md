**Helper: `withRetry<T>(fn, maxRetries=3, delays=[1000,5000,15000])`**
Simple retry wrapper with exponential backoff for EmailBison API calls.

**Export `executeDeploy(campaignId: string, deployId: string): Promise<void>`**

This is the core fire-and-forget function called after the API response is sent.

Logic:
1. Update CampaignDeploy status → 'running'
2. Load campaign via getCampaign(campaignId). Validate it exists and has status 'deployed'.
3. Load workspace (for apiToken to create EmailBisonClient)
4. Parse channels from campaign (JSON array)
5. Load leads from TargetList:
   ```typescript
   const leads = await prisma.targetListPerson.findMany({
     where: { listId: campaign.targetListId! },
     include: {
       person: {
         include: { workspaces: { where: { workspace: campaign.workspaceSlug } } }
       }
     }
   });
   ```

6. **Email channel** (if channels includes 'email'):
   a. Set emailStatus → 'running'
   b. Create EB campaign: `withRetry(() => ebClient.createCampaign({ name: campaign.name }))`
   c. Store emailBisonCampaignId on both CampaignDeploy and Campaign records
   d. Create sequence steps from campaign.emailSequence (parsed JSON array):
      For each step: `withRetry(() => ebClient.createSequenceStep(ebCampaignId, { position: step.position, subject: step.subjectLine, body: step.body ?? step.bodyText, delay_days: step.delayDays ?? 1 }))`
   e. Push leads (serial with 100ms delay between):
      - Dedup: skip if Person already has a WebhookEvent with eventType='EMAIL_SENT' for this workspace (Outsignal-side dedup)
      - For each lead: `withRetry(() => ebClient.createLead({ email: person.email, firstName: person.firstName, lastName: person.lastName, jobTitle: person.jobTitle, company: person.company }))`
      - Track count of successfully pushed leads
   f. Update CampaignDeploy: emailStatus → 'complete', emailStepCount, leadCount
   g. On error: set emailStatus → 'failed', emailError = error message

7. **LinkedIn channel** (if channels includes 'linkedin'):
   a. Set linkedinStatus → 'running'
   b. Parse campaign.linkedinSequence (JSON array of { position, type, body, delayDays })
   c. For each lead with a linkedinUrl:
      - Assign sender via `assignSenderForPerson(workspaceSlug, { mode: channels.includes('email') ? 'email_linkedin' : 'linkedin_only' })`
      - If no sender available: log warning and skip
      - For LinkedIn-only campaigns: schedule first step for each lead with staggered delays (lead index * 15 minutes between leads to avoid burst)
      - For email+linkedin campaigns: only enqueue the first linkedin step if triggerEvent is 'delay_after_previous' (not email_sent triggered). Email-triggered steps are handled by the webhook (Plan 03).
      - `enqueueAction({ senderId, personId, workspaceSlug, actionType: step.type, messageBody: step.body, priority: 5, scheduledFor: computedDate, campaignName: campaign.name, sequenceStepRef: `linkedin_${step.position}` })`
   d. Update CampaignDeploy: linkedinStatus → 'complete', linkedinStepCount
   e. On error: set linkedinStatus → 'failed', linkedinError = error message

8. **Finalize:**
   - If both channels complete: status → 'complete'
   - If one channel failed: status → 'partial_failure'
   - If all channels failed: status → 'failed'
   - Set completedAt = new Date()

**Export `retryDeployChannel(deployId: string, channel: 'email' | 'linkedin'): Promise<void>`**
Loads the CampaignDeploy, sets retryChannel, resets the failed channel status, and re-runs just that channel's logic from executeDeploy.
