  <action>
Add three new methods to the EmailBisonClient class in `src/lib/emailbison/client.ts`:

1. `getReply(replyId: number): Promise<Reply>` — Fetch a single reply by ID.
   - Call `this.request<{ data: Reply }>(\`/replies/${replyId}\`, { revalidate: 0 })`
   - Return `res.data`
   - revalidate: 0 is required for real-time reads (bypass Next.js fetch cache)

2. `getRepliesPage(page: number = 1): Promise<PaginatedResponse<Reply>>` — Fetch one page of replies (for inbox thread list pagination).
   - Call `this.request<PaginatedResponse<Reply>>(\`/replies?page=${page}\`, { revalidate: 0 })`
   - Return the full PaginatedResponse (caller controls pagination)
   - This is different from existing getReplies()/getRecentReplies() which auto-paginate

3. `sendReply(replyId: number, params: SendReplyParams): Promise<SendReplyResponse>` — Send a reply to an existing reply thread.
   - Call `this.request<SendReplyResponse>(\`/replies/${replyId}/reply\`, { method: 'POST', body: JSON.stringify(params), revalidate: 0 })`
   - Follow the same pattern as createCampaign() and createLead()

Import SendReplyParams and SendReplyResponse from ./types (add to existing import statement at top of file).

Do NOT modify existing methods. Do NOT change the import list except to add the 2 new type imports.
