```typescript
export interface VoyagerConversation {
  entityUrn: string;           // LinkedIn's entityUrn for the conversation
  conversationId: string;      // Extracted ID for use in messages fetch
  participantUrns: string[];   // URNs of participants
  participantNames: string[];  // Display names where available
  lastActivityAt: number;      // Epoch ms from lastActivityAt
  snippet: string | null;      // Last message preview text
}

export interface VoyagerMessage {
  eventUrn: string;            // LinkedIn's event entityUrn (unique ID)
  senderUrn: string;           // Who sent this message
  senderName: string | null;   // Display name if available
  body: string;                // Message text content
  deliveredAt: number;         // Epoch ms timestamp
}
```

**New methods on VoyagerClient class:**

1. `async fetchConversations(limit: number = 20): Promise<VoyagerConversation[]>`
   - Call `this.request(\`/voyagerMessagingDashMessengerConversations?keyVersion=LEGACY_INBOX&q=all&count=${limit}\`)`
   - Check for checkpoint/challenge in response.url — throw VoyagerError(403, "checkpoint_detected") if detected
   - Parse the response JSON (LinkedIn normalized format)
   - Call a private `parseConversations(data)` method to extract conversation objects from the normalized response
   - The normalized response has `included[]` array with mixed entity types. Conversations have `$type` containing "MessengerConversation". Extract entityUrn, participants, lastActivityAt, and snippet from each.
   - For conversationId: extract from entityUrn — the ID portion after the last colon
   - CRITICAL: Log `JSON.stringify(data).slice(0, 3000)` to console during first run to capture actual response shape. The parser will likely need adjustment based on real data. Write the parser as best-effort with defensive checks (null-coalescing, optional chaining) so it returns empty arrays rather than crashing on unexpected shapes.
   - Wrap in try/catch — on VoyagerError rethrow, on other errors throw new VoyagerError(0, String(err))

2. `async fetchMessages(conversationId: string, count: number = 20): Promise<VoyagerMessage[]>`
   - Call `this.request(\`/voyagerMessagingDashMessengerMessages?conversationUrn=urn%3Ali%3AmessagingThread%3A${conversationId}&count=${count}\`)`
   - Note: the conversationUrn parameter needs URL encoding. The format is `urn:li:messagingThread:{conversationId}`.
   - If that path 404s, try alternative: `/messaging/conversations/${conversationId}/events?count=${count}`
   - Check for checkpoint/challenge — throw VoyagerError(403, "checkpoint_detected") if detected
   - Parse normalized JSON response. Messages have `$type` containing "MessengerMessage" or similar.
   - Extract eventUrn, sender URN, body text, and deliveredAt timestamp from each message
   - Log raw response on first call for debugging (same pattern as fetchConversations)
   - Wrap in try/catch — same error handling as fetchConversations

3. `private parseConversations(data: Record<string, unknown>): VoyagerConversation[]`
   - Look in `data.included` (array) for entities with `$type` matching conversation type
   - Also check `data.data?.elements` or `data.data?.["*elements"]` as the response format varies
   - Extract conversation fields with defensive null checks
   - Return empty array if structure doesn't match expectations (don't crash)

4. `private parseMessages(data: Record<string, unknown>): VoyagerMessage[]`
   - Same approach as parseConversations but for message entities
   - Look for message body in `body.text` or `body.attributes[0].text` or direct `body` field
   - Extract deliveredAt from epoch ms field
   - Return empty array on unexpected structure

IMPORTANT: Do NOT modify the existing `request()` method. Do NOT modify any existing methods. Only add new types and methods. The Voyager API response shape is only medium-confidence from research — the parser should be defensive and log raw data.
