**popup.js** — Vanilla JS, no framework, no build step:

Constants at top:
```javascript
const API_BASE = 'https://admin.outsignal.ai';
```

State management via chrome.storage.local. Keys:
- `apiToken` — sender-scoped token (Bearer)
- `workspaceToken` — workspace-scoped token (before sender selection)
- `selectedSenderId` — current sender ID
- `workspaceSlug` — for display

On popup open (DOMContentLoaded):
1. Check chrome.storage.local for `apiToken` and `selectedSenderId`
2. If both exist, call GET /api/extension/status to validate token and get current status
3. If status returns 401, clear storage, show login view
4. If status returns valid sender, show status view with appropriate indicator
5. If no token, show login view

Login form submit handler:
1. Disable button, show "Logging in..."
2. POST /api/extension/login with { email, workspaceSlug, password }
3. On success:
   - If senderToken returned (single sender): store apiToken + selectedSenderId + workspaceSlug in chrome.storage.local, show status view
   - If multiple senders: store workspaceToken + workspaceSlug, populate sender dropdown, show sender-view
4. On error: show error message

Sender selection handler:
1. POST /api/extension/select-sender with Bearer workspaceToken + { senderId }
2. Store apiToken (= senderToken) + selectedSenderId in chrome.storage.local
3. Show status view

Connect button handler:
1. Disable button, show "Connecting..."
2. Call `chrome.cookies.getAll({ domain: '.linkedin.com' })`
3. If no cookies found, show error "Please log into LinkedIn first in your browser"
4. Map cookies to `{ name, value, domain }` objects
5. POST /api/extension/senders/{selectedSenderId}/cookies with Bearer apiToken
6. On success: update status indicator to green checkmark, show toast "LinkedIn connected", clear badge
7. On error: show toast with error message

Status view rendering:
- Green circle with checkmark + "LinkedIn Connected" + last active time — if sessionStatus === "active"
- Red circle with X + "Session Expired" + "Reconnect" button — if sessionStatus === "expired" or healthStatus === "session_expired"
- Gray circle + "Not Connected" + "Connect LinkedIn" button — if sessionStatus === "not_setup"
- Connect/Reconnect button always visible (same handler — reads cookies and POSTs)

Logout handler:
- Clear chrome.storage.local (apiToken, workspaceToken, selectedSenderId, workspaceSlug)
- Show login view
- Clear badge text

Toast helper:
```javascript
function showToast(message, type = 'success') {
  const toast = document.createElement('div');
  toast.className = `toast toast-${type}`;
  toast.textContent = message;
  document.body.appendChild(toast);
  setTimeout(() => { toast.style.opacity = '0'; setTimeout(() => toast.remove(), 300); }, 3000);
}
```

Badge management (communicate with service worker):
```javascript
// Clear badge on successful connect
chrome.action.setBadgeText({ text: '' });
```

Error handling: all fetch calls wrapped in try/catch. Network errors show "Connection failed — check your internet" toast.
