  <name>Task 1: Create nova-memory seed script with per-workspace memory generation</name>
  <files>scripts/nova-memory.ts</files>
  <action>
Create `scripts/nova-memory.ts` -- a CLI script runnable via `npx tsx scripts/nova-memory.ts seed [slug|--all]`.

**CLI interface:**
- `seed {slug}` -- seed one workspace
- `seed --all` -- seed all workspaces from DB (query `prisma.workspace.findMany`)
- No args or invalid command -- print usage help

**Per-workspace seeding (`seedWorkspace(slug)`):**

1. Create directory `.nova/memory/{slug}/` with `fs.mkdir({ recursive: true })`
2. Create `.gitkeep` in the slug directory (empty file for git tracking)

3. **profile.md** (ALWAYS overwrite -- fresh from DB):
   - Query workspace with non-sensitive fields only (see interfaces above -- never select apiToken, clientEmails, billingRetainerPence, billingCompanyName, etc.)
   - Governance header: `<!-- profile.md | workspace: {slug} | seeded: {ISO date} | re-seed: overwrites this file -->`
   - Second line: `<!-- Write governance: NEVER append to this file — it is regenerated by nova-memory seed -->`
   - Sections: Company (name, vertical, website, package, enabled modules, monthly campaign allowance, monthly lead quota), ICP (icpCriteriaPrompt or placeholder "Not configured"), Tone & Voice (outreachTonePrompt or placeholder), Outreach Channels (derived from enabledModules JSON), Key Contacts (senderFullName, senderJobTitle or placeholder), Standing Instructions (empty section with HTML comment explaining admin-editable rules)
   - Use empty-string fallbacks with explicit placeholder text for sparse workspaces (e.g. ladder-group) -- never output `undefined` or `null`

4. **campaigns.md** (skip if file exists -- preserve accumulated data):
   - Governance header: `<!-- campaigns.md | workspace: {slug} | seeded: {ISO date} | re-seed: skips if exists -->`
   - Second line: `<!-- Write governance: APPEND only, with ISO timestamp. Max 200 lines. Never delete existing entries. -->`
   - Query campaigns for this workspace: `prisma.campaign.findMany({ where: { workspaceSlug: slug } })`
   - Query CachedMetrics snapshots: `prisma.cachedMetrics.findMany({ where: { metricType: 'campaign_snapshot', metricKey: { in: campaignIds } } })`
   - Merge into markdown table: Campaign | Channel | Status | Reply Rate | Open Rate | Leads
   - Parse `CachedMetrics.data` with `JSON.parse(s.data as string)` wrapped in try/catch
   - Parse `campaign.channels` with `JSON.parse(c.channels || '[]')`
   - Add empty "Copy Wins" and "Copy Losses" sections with agent instruction comments

5. **feedback.md** (skip if file exists):
   - Governance header with append-only rule
   - Empty "Approval History" section with comment: `<!-- Agent: append entries as: [ISO date] Campaign: [name] — [Approved|Rejected] — [reason if known] -->`
   - Empty "Client Preferences Observed" section

6. **learnings.md** (skip if file exists):
   - Governance header with append-only rule
   - Empty sections: ICP Learnings, Lead Source Effectiveness, Vertical-Specific Insights (using workspace vertical in section name)

**Global insights seeding (`seedGlobalInsights()`):**

1. Always overwrite `.nova/memory/global-insights.md`
2. Governance header: `<!-- global-insights.md | seeded: {ISO date} | writer: nova-intel ONLY -->`
3. Second line: `<!-- Write governance: ONLY nova-intel agent may write here. All other agents: READ ONLY. APPEND with timestamp. Max 200 lines. -->`
4. Query ALL campaign snapshots: `prisma.cachedMetrics.findMany({ where: { metricType: 'campaign_snapshot' } })`
5. Parse each, filter to active/completed campaigns with totalLeadsContacted > 50
6. Compute cross-client averages: avg reply rate, avg open rate, avg bounce rate for email campaigns
7. Group by workspace vertical if available (from workspace query)
8. Output sections: Cross-Client Benchmarks (table), Best Performing Channels, Copy Patterns (empty placeholder), Operational Insights (empty placeholder)

**Create `.nova/memory/.gitkeep`** (root-level gitkeep for the memory directory itself)

**Console output:** Print progress for each workspace: `Seeding {slug}... profile.md (overwritten), campaigns.md (skipped/created), feedback.md (skipped/created), learnings.md (skipped/created)`

**Important:** Import PrismaClient from the generated client (`const { PrismaClient } = require('@prisma/client')` or `import { PrismaClient } from '@prisma/client'`). Do NOT use path aliases (@/) -- use relative imports only, consistent with other scripts in the scripts/ directory.

<task type="auto">
  <name>Task 2: Configure gitignore for memory files</name>
  <files>.gitignore</files>
  <action>
Add the following entry to `.gitignore` (append at the end, with a comment header):

```
# Nova agent memory — gitignored to prevent client intelligence leaking to version control
# Directory structure preserved via .gitkeep files (which have no .md extension)
.nova/memory/**/*.md
```

This pattern ignores all `.md` files in `.nova/memory/` subdirectories (profile.md, campaigns.md, feedback.md, learnings.md, global-insights.md) while preserving `.gitkeep` files since they have no extension.

No negation pattern is needed -- `.gitkeep` files don't match `**/*.md`.
  </action>
