1. Create src/lib/insights/generate.ts:
   - Export async function generateInsights(workspaceSlug: string): Promise<number> — returns count of insights created
   - Gather all analytics data for the workspace (these are all pre-computed, no heavy queries):
     a. CachedMetrics where workspace = slug, metricType = "campaign_snapshot", order by date DESC, limit to last 2 weeks of snapshots. Parse each data JSON to get campaign performance (sent, opened, replied, bounced, interested, replyRate, openRate, bounceRate, interestedRate, copyStrategy).
     b. Reply stats: groupBy intent (count per intent), groupBy sentiment (count per sentiment), groupBy objectionSubtype where intent="objection" (count per subtype). Filter to last 2 weeks.
     c. CachedMetrics where metricType = "body_elements" for copy element data (if any exist).
     d. Current week vs prior week comparison: compute replyRate/openRate/interestedRate for this week vs last week.
   - Build a structured prompt with all gathered data. Tone: "You are an outbound email campaign analyst. Analyze this workspace data and produce 3-5 actionable insights. Be direct and data-first — lead with numbers, not fluff."
   - Include in prompt: campaign performance trends, intent distribution shifts, objection patterns, week-over-week changes. Tell the LLM the 4 available action types with descriptions of what each does.
   - Confidence thresholds guidance in prompt: HIGH = 50+ data points + >20% change, MEDIUM = 20-49 data points or 10-20% change, LOW = <20 data points or <10% change.
   - Call generateObject with anthropic("claude-haiku-4-5-20251001"), InsightSchema from types.ts.
   - Do NOT pass maxTokens (AI SDK limitation per decision [24-01]).
   - After generation, call filterDuplicates from dedup.ts to remove already-dismissed insights.
   - For remaining insights, prisma.insight.createMany with all fields populated. Set dedupKey using buildDedupKey(category, actionType, first relevant entity ID from params or "global").
   - Return count of created insights.
