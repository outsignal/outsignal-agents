     d. If execution fails: update status="failed", executionResult=JSON.stringify({error: message}).

<task type="auto">
  <name>Task 2: Action executors with audit trail</name>
  <files>src/lib/insights/actions.ts</files>
  <action>
1. Create src/lib/insights/actions.ts:
   - Export async function executeAction(insight: Insight from Prisma): Promise<{before: string, after: string, outcome: string}>
   - Switch on insight.actionType:

   a. "pause_campaign":
      - Parse actionParams to get campaignId.
      - Find campaign: prisma.campaign.findUnique({where: {id: campaignId}}).
      - If not found, throw "Campaign not found".
      - Record before = campaign.status (or campaign.ebStatus).
      - Update local status: prisma.campaign.update({where: {id}, data: {status: "paused"}}).
      - Note: EmailBison API may not support remote pause. Document this: "Campaign paused locally. Admin should verify pause in EmailBison dashboard."
      - Return {before: `Status: ${before}`, after: "Status: paused", outcome: "Campaign paused locally. Verify in EmailBison."}.

   b. "update_icp_threshold":
      - Parse actionParams to get workspaceSlug and newThreshold.
      - Find workspace: prisma.workspace.findUnique.
      - Record before = current icpCriteria JSON or icpCriteriaPrompt (whatever stores the threshold).
      - Note: ICP threshold is part of the workspace's icpCriteriaPrompt (a text prompt, not a numeric field). Since there's no dedicated numeric threshold field, this action should update a note/flag rather than modify the prompt directly. Create a CachedMetrics entry with metricType "icp_threshold_recommendation" storing the recommended value, so the admin can see it.
      - Actually simpler: store the recommendation as a note in executionResult. The insight itself IS the recommendation. Mark as executed.
      - Return {before: "No threshold set", after: `Recommended threshold: ${newThreshold}`, outcome: "Recommendation recorded. Admin should update ICP criteria prompt."}.

   c. "flag_copy_review":
      - Parse actionParams to get campaignId (or campaignName).
      - This is informational — flag the campaign for review. No destructive action.
      - Store a CachedMetrics entry with metricType "copy_review_flag", metricKey = campaignId, data = JSON with reason.
      - Return {before: "Not flagged", after: "Flagged for copy review", outcome: "Campaign flagged. Review copy in analytics."}.

   d. "adjust_signal_targeting":
      - Parse actionParams to get campaignId and suggested adjustment.
      - Similar to ICP threshold — this is a recommendation, not an auto-execute.
      - Return {before: "Current targeting", after: `Recommended: ${adjustment}`, outcome: "Signal targeting recommendation recorded."}.

   - All action types should wrap execution in try/catch and return meaningful error on failure.
  </action>
  <verify>node -e "const a = require('./src/lib/insights/actions'); console.log(typeof a.executeAction === 'function' ? 'OK' : 'FAIL')"</verify>
  <done>executeAction handles all 4 action types with before/after tracking. pause_campaign updates local campaign status. Other actions record recommendations. All results stored as executionResult JSON on the Insight.</done>
</task>
