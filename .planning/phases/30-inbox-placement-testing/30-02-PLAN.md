1. Create src/app/api/placement-tests/route.ts with maxDuration = 60 (Vercel timeout).

2. POST handler — trigger a placement test:
   - Auth: require admin session (use existing auth pattern from other API routes — check for valid session).
   - Request body: { senderEmail: string, workspaceSlug: string }
   - Validation: senderEmail and workspaceSlug required.
   - Check MAILTESTER_API_KEY exists — if not, return 503 with { error: "Placement testing not configured — MAILTESTER_API_KEY not set" }.
   - Look up the sender_email_id from EmailBison: use EmailBisonClient(workspaceToken).getSenderEmails(), find matching email. If not found, return 404.
   - Get workspace token: query Workspace model for the workspace's emailBisonToken.
   - Call getTestAddress(apiKey) to get mail-tester test address + testId.
   - Create PlacementTest record with status "pending", testAddress, senderEmail, senderDomain (derive from email), workspaceSlug.
   - Call sendTestEmail({ senderEmailId, toAddress: testAddress, workspaceToken }) to send the test email via EmailBison dedi.
   - If send fails: update PlacementTest status to "failed", return 500 with error.
   - Call pollForResults(testId, apiKey) — polls up to 60s.
   - If results returned: update PlacementTest with score, details (full JSON), status "completed", completedAt. Classify score. If warning or critical: upsert EmailSenderHealth record (set emailHealthStatus, lastTestScore, lastTestAt, statusReason). Call notifyPlacementResult().
   - If results still pending after polling: leave status as "pending", return 202 with { status: "pending", testId, message: "Results not yet ready. Re-fetch via GET." }.
   - On success: return 200 with the PlacementTest record.

3. GET handler — retrieve test history and re-fetch pending:
   - Auth: require admin session.
   - Query params: senderEmail (required), limit (optional, default 10).
   - If ?refetch=true and there are pending tests: for each pending test, attempt fetchTestResults. If results now available, update record (score, details, status, completedAt), classify, update EmailSenderHealth, notify.
   - Return PlacementTest records ordered by createdAt desc.
   - Include a `recommended` boolean field in response indicating if this sender is recommended for testing (call isRecommendedForTesting).

4. Error handling: wrap entire handlers in try/catch, return structured { error } responses.
