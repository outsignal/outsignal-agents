**Conversation list route — GET /api/portal/inbox/linkedin/conversations:**

1. Auth via `getPortalSession()` → extract `workspaceSlug`. Return 401 on failure.
2. Query `prisma.linkedInConversation.findMany({ where: { workspaceSlug }, orderBy: { lastActivityAt: 'desc' }, take: 50 })`.
3. Two-query pattern for Person data (NO `include: { person: true }` — personId has no @relation to Person):
   - Collect non-null `personId` values into an array
   - `prisma.person.findMany({ where: { id: { in: personIds } }, select: { id: true, jobTitle: true, company: true } })`
   - Build a `Map<string, { jobTitle, company }>` for O(1) lookup
4. Map conversations to response shape:
   ```
   {
     id: conv.id,                    // internal cuid — used as URL param in UI
     conversationId: conv.conversationId,  // LinkedIn's ID — used for worker calls
     participantName: conv.participantName,
     participantHeadline: conv.participantHeadline,
     participantProfilePicUrl: conv.participantProfilePicUrl,
     lastMessageSnippet: conv.lastMessageSnippet,
     lastActivityAt: conv.lastActivityAt.toISOString(),
     unreadCount: conv.unreadCount,
     jobTitle: person?.jobTitle ?? conv.participantHeadline ?? null,  // fallback to headline
     company: person?.company ?? null,
     senderId: conv.senderId,
   }
   ```
5. Return `NextResponse.json({ conversations: result })`.

**Message fetch route — GET /api/portal/inbox/linkedin/conversations/[conversationId]/messages:**

The `conversationId` URL param is the internal cuid (`LinkedInConversation.id`), NOT LinkedIn's conversation ID.

1. Auth via `getPortalSession()` → extract `workspaceSlug`. Return 401 on failure.
2. Resolve conversation: `prisma.linkedInConversation.findFirst({ where: { id: params.conversationId, workspaceSlug } })`. Return 404 if not found.
3. Check for existing messages in DB: `prisma.linkedInMessage.findMany({ where: { conversationId: conv.id }, orderBy: { deliveredAt: 'asc' } })`.
4. If messages exist in DB AND no `?refresh=true` query param: return DB messages immediately.
5. If no messages OR `?refresh=true`: fetch from Railway worker:
   - Read `LINKEDIN_WORKER_URL` and `WORKER_API_SECRET` from env
   - `fetch(${WORKER_URL}/sessions/${conv.senderId}/conversations/${conv.conversationId}/messages, { headers: { Authorization: Bearer ${WORKER_SECRET} } })`
   - Defensive parse: `const raw = await res.json(); const workerMessages = raw?.messages ?? raw?.data ?? [];`
   - Log if neither key found: `console.warn('[linkedin-messages] Unknown response envelope:', Object.keys(raw))`
   - For each worker message, upsert into LinkedInMessage:
     ```typescript
     await prisma.linkedInMessage.upsert({
       where: { eventUrn: msg.eventUrn },
       create: {
         conversationId: conv.id,  // internal cuid FK
         eventUrn: msg.eventUrn,
         senderUrn: msg.senderUrn,
         senderName: msg.senderName,
         body: msg.body,
         isOutbound: msg.senderUrn !== conv.participantUrn,
         deliveredAt: new Date(msg.deliveredAt),
       },
       update: {},  // messages immutable
     });
     ```
   - After upsert, re-query DB for consistent ordering: `prisma.linkedInMessage.findMany({ where: { conversationId: conv.id }, orderBy: { deliveredAt: 'asc' } })`.
   - If worker fetch fails (non-200 or network error), log the error and return existing DB messages (graceful degradation).
6. Return `NextResponse.json({ messages, conversationId: conv.id, participantName: conv.participantName, participantUrn: conv.participantUrn, senderId: conv.senderId })`.

**Reply queue route — POST /api/portal/inbox/linkedin/reply:**

1. Auth via `getPortalSession()` → extract `workspaceSlug`. Return 401 on failure.
2. Parse request body: `const { conversationId, message } = await request.json()` — both required strings.
3. Validate: return 400 if `conversationId` or `message` is missing/empty.
4. Resolve conversation: `prisma.linkedInConversation.findFirst({ where: { id: conversationId, workspaceSlug }, select: { senderId: true, personId: true } })`.
5. If not found: return 404 `{ error: "Conversation not found" }`.
6. If `conv.personId` is null: return 422 `{ error: "Cannot queue reply: lead not matched to a Person record" }`.
7. Call `enqueueAction()` from `@/lib/linkedin/queue`:
   ```typescript
   import { enqueueAction } from "@/lib/linkedin/queue";

   const actionId = await enqueueAction({
     senderId: conv.senderId,
     personId: conv.personId,
     workspaceSlug,
     actionType: "message",
     messageBody: message,
     priority: 1,              // warm lead fast-track
     scheduledFor: new Date(),  // immediate
   });
   ```
8. Return `NextResponse.json({ actionId }, { status: 201 })`.

**Action status route — GET /api/portal/inbox/linkedin/actions/[actionId]/status:**

1. Auth via `getPortalSession()` → extract `workspaceSlug`. Return 401 on failure.
2. Query: `prisma.linkedInAction.findFirst({ where: { id: params.actionId, workspaceSlug }, select: { status: true, completedAt: true } })`.
3. If not found: return 404 `{ error: "Action not found" }`.
4. Return `NextResponse.json({ status: action.status, completedAt: action.completedAt?.toISOString() ?? null })`.
