Create `src/lib/discovery/validation.ts` as a pure-function validation module following the copy-quality.ts pattern (no side effects, composable checks, exported types).

**Types to export:**

```typescript
export interface ValidationIssue {
  type: "hard-block" | "warning";
  check: "company-name-vs-domain" | "missing-icp-fields" | "filter-platform-mismatch" | "budget-exceeded" | "icp-mismatch";
  message: string;
  suggestion: string;
}

export interface ValidationResult {
  valid: boolean;        // false if any hard-blocks
  issues: ValidationIssue[];
}
```

**Main function to export:**

```typescript
export function validateDiscoveryFilters(
  source: string,  // "apollo" | "prospeo" | "aiark" | "leads-finder" | "google-maps" | "ecommerce-stores"
  filters: Record<string, unknown>,
  context?: {
    workspaceIcp?: Record<string, unknown>;
    estimatedCostUsd?: number;
    remainingBudgetUsd?: number;
  }
): ValidationResult
```

**Implement 5 checks in order:**

1. **Company Name vs Domain** (hard-block):
   - If `filters.companyDomains` is an array, check each entry
   - Entry without a dot (`.`) -> hard-block: "'{entry}' looks like a company name, not a domain"
   - Entry with spaces -> hard-block: "'{entry}' contains spaces -- use domains like 'acme.com', not company names"
   - Suggestion: "Use company domains like 'acme.com', not company names like 'Acme Corp'"

2. **Missing Required ICP Fields** (hard-block):
   - If ALL of these are empty/missing: jobTitles, seniority, industries, companyDomains
   - Hard-block: "Too broad -- at least one of: job titles, seniority, industries, or company domains required"
   - Suggestion: "Add at least one targeting filter to narrow the search"
   - EXCEPTION: google-maps and ecommerce-stores are company-discovery tools that use different filter sets (categories, keywords, locations) -- skip this check for those two sources

3. **Filter-Platform Mismatch** (hard-block for broken filters, warning for unsupported):
   - AI Ark + departments -> hard-block: "AI Ark contact.department filter is broken (silently ignores filter, returns all records)"
   - AI Ark + keywords (contact-level, not companyKeywords) -> hard-block: "AI Ark contact.keyword returns 400 error"
   - Apollo + sicCodes -> warning: "Apollo does not support SIC codes -- use Prospeo instead"
   - Apollo + yearsExperience -> warning: "Apollo does not support years of experience -- use Prospeo"
   - Apollo + fundingStages -> warning: "Apollo free tier has limited funding filter support"

4. **Budget Exceeded** (warning):
   - Only if `context.estimatedCostUsd` and `context.remainingBudgetUsd` are provided
   - If estimated > remaining -> warning: "Estimated cost ${estimated} would exceed remaining daily budget of ${remaining}"
   - Suggestion: "Reduce page count or remove a paid source"

5. **ICP Mismatch** (warning):
   - Only if `context.workspaceIcp` is provided
   - Compare search industries vs workspace ICP industries -- if no overlap, warning
   - Compare search locations vs workspace ICP geographies -- if no overlap, warning
   - These are informational warnings only

`valid` is `false` if ANY issue has `type: "hard-block"`. Warnings do not affect `valid`.

Import `PROVIDER_COSTS` from `@/lib/enrichment/costs` for reference in budget check comments/documentation, but the actual budget values come from the `context` parameter (the caller computes them).

Do NOT use any async operations or database calls -- this module must be pure and synchronous. The caller (CLI wrapper) handles async context gathering if needed.

2. After parsing params from the JSON file, BEFORE calling leadsTools.search*.execute(), add:

```typescript
const validation = validateDiscoveryFilters("{source-name}", params);
if (!validation.valid) {
  const blocks = validation.issues.filter(i => i.type === "hard-block");
  console.error(`\nBLOCKED (${blocks.length} issue${blocks.length > 1 ? "s" : ""}):`);
  for (const issue of blocks) {
    console.error(`  [${issue.check}] ${issue.message}`);
    console.error(`  Suggestion: ${issue.suggestion}`);
  }
  process.exit(1);
}
// Log warnings (don't block)
const warnings = validation.issues.filter(i => i.type === "warning");
if (warnings.length) {
  console.warn(`\nWARNINGS (${warnings.length}):`);
  for (const w of warnings) {
    console.warn(`  [${w.check}] ${w.message}`);
  }
}
```
