  <name>Task 1: Add memory read functions to memory.ts</name>
  <files>src/lib/agents/memory.ts</files>
  <action>
Add the following functions to memory.ts (keep existing `appendToMemory` unchanged):

1. **`readMemoryFile(filePath: string, maxLines = 200): Promise<string | null>`**
   - Read file with `readFile` (async, NOT sync). Return null if file missing (try/catch on access).
   - If lines exceed maxLines: keep first 3 lines (header/comments) + `<!-- truncated: older entries removed -->` + last (maxLines - 4) lines. Log warning: `[memory] Truncating ${filePath}: ${lines.length} lines -> ${maxLines}`.
   - Return trimmed content or null if empty string.

2. **`hasRealEntries(content: string): boolean`**
   - Check if content contains real timestamped entries: `/\[\d{4}-\d{2}-\d{2}T/.test(content)`.
   - Seed-only files contain "(No X recorded yet)" and HTML comments but no ISO-dated entries. Return false for these.

3. **`loadSystemContext(): Promise<string | null>`**
   - Resolve path: `join(homedir(), ".claude/projects/-Users-jjay-programs/memory/MEMORY.md")` using `os.homedir()`.
   - Call `readMemoryFile(path)`. This will be null in production (Vercel) -- that's expected.
   - Per user decision: load the ENTIRE file, not condensed. The 200-line truncation still applies as a safety net.

4. **`loadCrossClientContext(): Promise<string | null>`**
   - Path: `join(projectRoot, MEMORY_ROOT, "global-insights.md")` where `projectRoot = process.env.PROJECT_ROOT ?? process.cwd()`.
   - Call `readMemoryFile(path)`. Check `hasRealEntries()` -- if seed-only, return null.

5. **`loadWorkspaceMemory(slug: string): Promise<string | null>`**
   - Read 4 files from `.nova/memory/{slug}/`: `profile.md`, `learnings.md`, `campaigns.md`, `feedback.md`.
   - For each file: call `readMemoryFile()`, then check `hasRealEntries()` -- skip seed-only files.
   - Join non-null results with `\n\n---\n\n`. Return null if all files are empty/seed-only.

6. **`formatMemoryContext(systemCtx, crossClientCtx, workspaceCtx): string`**
   - Use XML-style tags to delimit sections (as per RESEARCH.md Pattern 2):
     - `<system_memory>` for system context
     - `<cross_client_memory>` for cross-client patterns
     - `<workspace_memory>` for workspace history
   - Wrap all in `<agent_memory>` with instruction text: "The following is your persistent memory from previous sessions. Use it to inform your decisions but prioritize workspace-specific memory over cross-client patterns, and cross-client patterns over system state."
   - Return empty string if all three contexts are null.

7. **`loadMemoryContext(workspaceSlug?: string): Promise<string>`** (exported)
   - Call `loadSystemContext()`, `loadCrossClientContext()`, and optionally `loadWorkspaceMemory(slug)` in parallel via `Promise.all`.
   - Pass results to `formatMemoryContext()`.
   - Wrap entire function in try/catch -- on ANY error, log `[memory] Failed to load memory context:` and return empty string (graceful degradation).

Add `import { homedir } from "os"` at top of file. Export only `loadMemoryContext` (plus existing `appendToMemory`).
