Add an `isValidEntry()` guard function inside memory.ts (not exported) and call it before appending:

```typescript
function isValidEntry(entry: string): boolean {
  if (!entry || entry.trim().length === 0) return false;
  if (entry.includes("undefined: undefined")) return false;
  if (entry.includes("undefined --")) return false;
  if (entry.trim() === "undefined") return false;
  return true;
}
```

In `appendToMemory()`, add the validation check BEFORE the timestamp/append logic (after the max-lines check, before line 48):

```typescript
// Validate entry content
if (!isValidEntry(entry)) {
  console.warn(`[memory] Rejecting malformed entry for ${slug}/${file}: "${entry.slice(0, 50)}"`);
  return false;
}
```

This prevents future garbage writes without changing the function signature or return type. The function already returns `false` for various skip conditions (file missing, at capacity), so this is consistent.
