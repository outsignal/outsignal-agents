**Create src/app/api/documents/upload/route.ts:**
- POST endpoint supporting two modes via content-type detection:
  1. **PDF file upload (multipart/form-data)**: Accept FormData with a `file` field. Validate content type is `application/pdf`. Max size 10MB. Extract text using `pdf-parse`:
    ```typescript
    import pdfParse from "pdf-parse";
    const buffer = Buffer.from(await file.arrayBuffer());
    const parsed = await pdfParse(buffer);
    const text = parsed.text;
    ```
  2. **Google Doc URL (application/json)**: Accept JSON `{ url: string }`. Extract doc ID from URL, fetch export as plain text:
    ```typescript
    const docId = url.match(/\/d\/([a-zA-Z0-9_-]+)/)?.[1];
    const exportUrl = `https://docs.google.com/document/d/${docId}/export?format=txt`;
    const res = await fetch(exportUrl);
    if (!res.ok || !res.headers.get("content-type")?.includes("text/plain")) {
      return NextResponse.json({ error: "Document must be publicly shared" }, { status: 400 });
    }
    const text = await res.text();
    ```

- After extracting text (from either mode), run a heuristic field parser:
  - Look for patterns: "Client Name:", "Company:", "Email:", "Package:", "Setup Fee:", "Platform Cost:", "Retainer:", etc.
  - Return: `{ parsed: { clientName?, clientEmail?, companyOverview?, packageType?, setupFee?, platformCost?, retainerCost? }, raw: string }`
  - The `raw` field contains the full extracted text for user reference.

- **CRITICAL**: This route MUST use Node.js runtime (NOT Edge). Do NOT add `export const runtime = "edge"`. pdf-parse requires Node.js `fs` module.

- Error handling:
  - 400 for invalid file type, oversized file, invalid Google Doc URL, private Google Doc
  - 500 for pdf-parse failures with descriptive error message
