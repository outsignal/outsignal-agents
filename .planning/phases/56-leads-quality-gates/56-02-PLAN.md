  <action>
Create `src/lib/discovery/domain-resolver.ts` implementing the full resolution pipeline.

**Interfaces to export:**
- `ResolutionResult`: companyName (string), domain (string | null), source ("db" | "serper" | "failed"), httpVerified (boolean)
- `ResolutionSummary`: total (number), resolved (number), failed (number), failedCompanies (string[]), results (ResolutionResult[]), totalCostUsd (number)

**Functions to export:**

1. `verifyDomainLive(domain: string): Promise<boolean>` — HTTP HEAD request with 5-second timeout via fetch AbortController. Returns true for 2xx/3xx responses. Returns false for 4xx/5xx/timeout/network error. Check for redirects to known parking services (sedo.com, godaddy.com/parked, afternic.com, hugedomains.com) and return false for those.

2. `resolveCompanyDomain(companyName: string, icpContext: { location?: string; industry?: string }): Promise<ResolutionResult>` — The 4-step pipeline:
   - Step 1: DB lookup — `prisma.company.findFirst({ where: { name: { contains: companyName, mode: "insensitive" } }, select: { domain: true, name: true } })`. If found, return with source="db", httpVerified=true (already in DB = trusted).
   - Step 2: Serper contextual search — Build query: `${companyName} ${icpContext.location ?? ''} ${icpContext.industry ?? ''} official website`. Use `serperAdapter.webSearch(query, { num: 3 })`. Extract domain from the top organic result's link using `new URL(result.link).hostname`. Strip "www." prefix.
   - Step 3: HTTP verification — call verifyDomainLive on the extracted domain. If fails, try the second result.
   - Step 4: Persist — If verified, upsert to Company table: `prisma.company.upsert({ where: { domain }, update: { name: companyName }, create: { domain, name: companyName } })`.
   - If all steps fail, return source="failed", domain=null, httpVerified=false.

3. `resolveCompanyDomains(companies: string[], icpContext: { location?: string; industry?: string }): Promise<ResolutionSummary>` — Batch resolver. Uses Promise.allSettled with concurrency limit of 10 (use a simple semaphore pattern — don't add a dependency). Collects all ResolutionResults, computes summary stats, returns ResolutionSummary.

CRITICAL per CONTEXT.md: Always include ICP context in Serper query — "Acme Corp UK recruitment agency" not just "Acme Corp". This is the key to avoiding wrong-company resolution.

**Test file** `src/lib/discovery/__tests__/domain-resolver.test.ts`:
- Mock prisma.company.findFirst, prisma.company.upsert
- Mock serperAdapter.webSearch
- Mock global fetch for HTTP verification
- Test DB-hit path (found in DB, no Serper call)
- Test Serper-fallback path (not in DB, Serper returns result, HTTP verified)
- Test failed path (not in DB, Serper returns no results)
- Test HTTP verification: live domain (200), redirect to parking (false), timeout (false)
- Test batch resolution with concurrency (3+ companies, verify all resolved)
- Test ICP context is included in Serper query string
- Test parking domain detection (redirect to sedo.com)
- At least 10 test cases
