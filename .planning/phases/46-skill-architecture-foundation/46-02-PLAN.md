    **Step 1: Create `src/lib/agents/load-rules.ts`**

    Export a `loadRules(filename: string): string` function that:
    - Resolves the rules path using `__dirname` relative navigation: `join(__dirname, '..', '..', '..', '.claude', 'rules', filename)` — this works from `src/lib/agents/` to project root
    - Add `PROJECT_ROOT` env var as a fallback: `process.env.PROJECT_ROOT ?? join(__dirname, '..', '..', '..')` — needed for compiled `dist/cli/` scripts (Phase 48 concern)
    - Calls `readFileSync(rulesPath, 'utf-8')` inside the function body (NOT at module top level — deferred to invocation time)
    - On catch: `console.warn('[nova] Rules file not found: ${rulesPath}')` and returns empty string (graceful degradation, does not crash the agent)
    - Import from 'fs' and 'path' only (no npm dependencies)

    **Step 2: Create `.claude/rules/` directory and extract rules from existing agents**

    For each of the 4 existing agents, read the system prompt constant from the TypeScript source and extract the behavioral rules portion into a markdown file. The rules file should contain ONLY the behavioral/instructional content — not the identity preamble or tool definitions.

    Extraction targets:
    - `writer.ts`: Find the `WRITER_SYSTEM_PROMPT` or equivalent constant. Extract copy quality rules (hyphen bans, tone constraints), sequence structure defaults, signal-aware writing rules, and strategy-specific instructions. Save to `.claude/rules/writer-rules.md`.
    - `leads.ts`: Find the system prompt constant. Extract the discovery workflow rules (plan-approve-execute pattern), credit gating, source priority order, approval workflow. Save to `.claude/rules/leads-rules.md`.
    - `orchestrator.ts` / `campaign.ts`: Find the system prompt constants. Extract campaign workflow rules, delegation routing, signal campaign rules. Save to `.claude/rules/campaign-rules.md`.     - `research.ts`: Find the system prompt constant. Extract research process rules, ICP extraction patterns, website analysis rules. Save to `.claude/rules/research-rules.md`.

    Each rules file format:
    ```markdown

    {extracted behavioral rules content}
    ```

  <action>
    Create `.nova/ARCHITECTURE.md` (create `.nova/` directory if it doesn't exist). This document locks the dual-mode strategy as a project decision.

    Sections to include:

    **1. Overview**
    - Nova is the CLI agent team for Outsignal. It runs via Claude Code skills (zero API cost) with API agent fallback (controlled by `USE_CLI_AGENTS` env var).
    - Two modes: CLI skills (primary, uses Claude Code Max Plan) and API agents (fallback, uses Anthropic API at ~$15/MTok for Opus).

    **2. Dual-Mode Strategy (LOCKED DECISION)**
    - Shared rules files in `.claude/rules/` are the SINGLE SOURCE OF TRUTH for agent behavioral rules
    - CLI skills reference rules via `!` (file include) syntax in their `.md` files
    - API agents reference rules via `loadRules()` function from `src/lib/agents/load-rules.ts`
    - Both modes get identical behavioral rules — zero drift by design
    - If a rule changes, edit the `.claude/rules/*.md` file ONCE — both modes pick it up
    - Per-agent rules files: `writer-rules.md`, `leads-rules.md`, `campaign-rules.md`, `research-rules.md`, `deliverability-rules.md`, `onboarding-rules.md`, `intelligence-rules.md`
    - API agents refactored to read rules from `.claude/rules/` at prompt-build time (not keeping hardcoded prompts in TypeScript)

    **3. Secret Handling**
    - `.claudeignore` blocks `.env*`, build artifacts, and database files from Claude Code context
    - `sanitize-output.ts` strips credentials from CLI wrapper stdout using `[REDACTED:type]` format
    - PII (emails, names) is NOT stripped — agents need it to do their job (locked decision)

    **4. 200-Line Skill Content Budget**
    - Each skill file (`.claude/commands/nova-*.md`) must stay under 200 lines
    - Skill files contain: identity, tools list, invocation patterns, memory read/write instructions
    - Behavioral rules go in `.claude/rules/*.md` and are included via `!` syntax
    - Enforcement: comment at top of each skill file noting line count. Automated enforcement deferred to Phase 49.
    - If a rules file exceeds 200 lines, it can be split (e.g., `writer-copy-rules.md` + `writer-strategies.md`)

    **5. Skill Registry**
    Full list of 8 Nova skills with command names:
    - `/nova {slug}` — orchestrator, delegates to specialists
    - `/nova-writer {slug}` — copy writing
    - `/nova-research {slug}` — ICP and market research
    - `/nova-leads {slug}` — lead discovery
    - `/nova-campaign {slug}` — campaign management
    - `/nova-deliver {slug}` — deliverability monitoring
    - `/nova-onboard {slug}` — client onboarding
    - `/nova-intel {slug}` — analytics and benchmarking
    - No slug = workspace picker
