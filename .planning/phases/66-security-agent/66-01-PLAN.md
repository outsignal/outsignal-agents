<task type="auto">
  <name>Task 2: Create monty-security.ts agent module</name>
  <files>src/lib/agents/monty-security.ts</files>
  <action>
Create src/lib/agents/monty-security.ts by cloning the monty-qa.ts pattern exactly. The file structure must be:

1. **Imports**: Same as monty-qa.ts (execSync from child_process, tool from ai SDK, z from zod, loadRules from local util if exists or inline, memory utilities from ./memory, types from ./types, AgentConfig from runner)

2. **runDevCli helper**: Copy from monty-qa.ts — wraps execSync calls to dev-cli scripts with PROJECT_ROOT and maxBuffer

3. **6 dev-cli tools** (copy from monty-qa.ts):
   - checkTypes — runs `npx tsc --noEmit`
   - readFile — reads a file by path
   - listFiles — lists files in a directory
   - searchCode — searches code with ripgrep pattern
   - gitDiff — shows recent git changes
   - runTests — runs vitest on specified paths

4. **1 new tool — npmAudit**:
   - Description: "Run npm audit and return vulnerability report with severity levels"
   - inputSchema: z.object({}) (no parameters)
   - Execute: runs `npm audit --json 2>/dev/null` via execSync
   - CRITICAL: npm audit exits with code 1 when vulnerabilities exist. execSync throws on non-zero exit. Must catch the error and parse error.stdout for the JSON output. Use the exact pattern from research file lines 113-133.

5. **System prompt** (MONTY_SECURITY_SYSTEM_PROMPT):
   Load the content of `.claude/rules/monty-security-rules.md` (use loadRules or inline readFileSync).
   
   The system prompt must include:
   - Role: Security gate for platform engineering changes
   - OWASP Top 10:2025 reference categories (map to the finding categories: A01 Broken Access Control -> auth-bypass, A02 Cryptographic Failures -> secrets-exposure, A03 Injection -> injection, A04 Insecure Design -> owasp-compliance, A05 Security Misconfiguration -> owasp-compliance, A06 Vulnerable Components -> dependency-vuln, A07 Auth Failures -> auth-bypass, A08 Data Integrity -> input-validation, A09 Logging Failures -> error-leakage, A10 SSRF -> injection)
   - Credential scanning guidance: regex patterns for API_KEY, SECRET, TOKEN, password, Bearer, hardcoded strings. Distinguish between hardcoded secrets (CRITICAL), process.env usage in server code (SAFE), env var names in client-side code (POTENTIAL ISSUE)
   - Auth flow review checklist from monty-security-rules.md
   - Output format: MUST return structured JSON matching MontySecurityOutput schema
   - blockDeploy rule: Set blockDeploy=true if ANY critical or high severity finding is unresolved. Include gateReason explaining why deployment is blocked.
   - NO minimum findings rule — only report real vulnerabilities. False positives erode trust.
   - Action tier: ALL security review work is Tier 1 (read-only). Security agent NEVER fixes code — it reports findings for the Dev Agent to fix.

6. **AgentConfig** (montySecurityConfig):
   - name: "monty-security"
   - systemPrompt: MONTY_SECURITY_SYSTEM_PROMPT
   - tools: { checkTypes, readFile, listFiles, searchCode, gitDiff, runTests, npmAudit }
   - outputSchema: montySecurityOutputSchema
   - maxSteps: 15
   - onComplete: async (output: MontySecurityOutput) => {
       // Write critical/high findings to .monty/memory/security.md
       const serious = output.findings.filter(f => f.severity === "critical" || f.severity === "high");
       if (serious.length > 0) {
         const lines = serious.map(f => `[${new Date().toISOString()}] [${f.severity.toUpperCase()}] ${f.file}:${f.line ?? "?"} — ${f.category}: ${f.description}`);
         await appendToMontyMemory("security.md", lines.join("\n"));
       }
       // Write cross-team notification to .nova/memory/global-insights.md
       if (output.affectsNova && output.novaNotification) {
         await appendToGlobalMemory(`[${new Date().toISOString()}] [Security] ${output.novaNotification}`);
       }
     }

7. **runMontySecurityAgent function**:
   - Takes MontySecurityInput, returns Promise<MontySecurityOutput>
   - Builds the user message from input.task + optional changedFiles list
   - Calls runAgent(montySecurityConfig, userMessage) — same pattern as runMontyQAAgent
   - Returns the parsed output

Check that MontyMemoryFile type in memory.ts includes "security.md". If it does NOT, add "security.md" to the union type. The research flagged this as a potential issue (Pitfall 4).

Export: runMontySecurityAgent, montySecurityConfig
