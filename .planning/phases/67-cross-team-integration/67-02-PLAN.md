Add a `getCrossTeamUpdates` function that:

1. **Reads both memory files:**
   - `.nova/memory/global-insights.md` (contains Monty-to-Nova entries among other content)
   - `.monty/memory/incidents.md` (contains Nova-to-Monty entries among other content)
   - Use `process.cwd()` as project root (same pattern as memory.ts)
   - If either file doesn't exist, skip it gracefully (return empty array for that direction)

2. **Parses cross-team entries from both files:**
   - Call `parseCrossTeamEntries(content)` on each file's content
   - Entries from global-insights.md will have direction "monty-to-nova"
   - Entries from incidents.md will have direction "nova-to-monty"

3. **Detects new entries since last poll using line-count tracking:**
   - Read a marker file at `.monty/memory/.last-cross-team-poll.json`
   - Marker format: `{ "globalInsightsLines": number, "incidentsLines": number, "lastPollTimestamp": string }`
   - If marker file doesn't exist, treat ALL cross-team entries as new (first poll)
   - "New" entries = entries beyond the last-known line count (compare total line count of file vs stored count)
   - After reading, update the marker file with current line counts and timestamp

4. **Returns a structured object:**
```typescript
interface CrossTeamResponse {
  montyToNova: CrossTeamEntry[];  // All Monty->Nova entries
  novaToMonty: CrossTeamEntry[];  // All Nova->Monty entries
  newEntries: CrossTeamEntry[];   // Only entries since last poll
  lastPollTimestamp: string | null;
  acknowledgmentInstructions: {
    montyToNova: string;  // "Run: npx tsx scripts/chat.ts to let Nova process these updates"
    novaToMonty: string;  // "Run: npx tsx scripts/monty.ts to let Monty triage these issues"
  };
}
```
