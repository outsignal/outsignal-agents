<objective>
Stop pre-scheduling post-connection messages and switch to event-driven follow-ups.

Purpose: Currently chainActions pre-schedules ALL LinkedIn sequence steps (including messages that should only fire after connection acceptance). This creates premature message actions that waste daily budget and produce incorrect analytics. The fix splits the sequence at the connection gate: pre-connect steps go to chainActions, post-connect steps become CampaignSequenceRules triggered by connection_accepted.

Output: Modified `chain.ts` and `deploy.ts` that implement the connection gate split.
</objective>
