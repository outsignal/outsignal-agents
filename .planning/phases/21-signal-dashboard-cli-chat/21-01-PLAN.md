  <name>Task 1: Create GET /api/signals aggregated endpoint</name>
  <files>src/app/api/signals/route.ts</files>
  <action>
Create `src/app/api/signals/route.ts` with a single GET handler that returns all data the dashboard needs in one request.

**Query parameters:**
- `workspace` — optional filter (default: "all" = no filter; specific slug = filter by workspaceSlug)
- `limit` — feed limit (default: 100, max: 500)

**Data to aggregate (run all queries in parallel via Promise.all):**

1. **Feed:** `prisma.signalEvent.findMany` ordered by `detectedAt desc`, take `limit`, select: id, signalType, companyName, companyDomain, workspaceSlug, title, isHighIntent, detectedAt, status. Apply workspace filter if provided.

2. **Type distribution:** `prisma.signalEvent.groupBy` by `signalType`, where `detectedAt >= sevenDaysAgo`, count each type. Apply workspace filter.

3. **Cost data:** `prisma.signalDailyCost.findMany` where `date >= sevenDaysAgoStr` (use `new Date().toISOString().slice(0, 10)` for UTC today, subtract 7 days for range start). Apply workspace filter. Order by date asc.

4. **Leads generated per workspace:** Since SignalCampaignLead does NOT have a direct `workspaceSlug` field, use this query:
```typescript
const leadsGenerated = await prisma.signalCampaignLead.findMany({
  where: {
    outcome: "added",
    addedAt: { gte: sevenDaysAgo },
    ...(workspaceFilter ? { campaign: { workspaceSlug: workspaceFilter } } : {}),
  },
  select: {
    campaign: { select: { workspaceSlug: true } },
  },
});
// Aggregate in JS:
const leadsByWorkspace: Record<string, number> = {};
for (const l of leadsGenerated) {
  const ws = l.campaign.workspaceSlug;
  leadsByWorkspace[ws] = (leadsByWorkspace[ws] ?? 0) + 1;
}
```

5. **Workspace caps:** `prisma.workspace.findMany` selecting `slug`, `name`, `signalDailyCapUsd` to show cap utilization. Apply workspace filter if provided.

**Date computation (all UTC):**
```typescript
const now = new Date();
const todayStr = now.toISOString().slice(0, 10);
const sevenDaysAgo = new Date(now);
sevenDaysAgo.setUTCDate(sevenDaysAgo.getUTCDate() - 7);
const sevenDaysAgoStr = sevenDaysAgo.toISOString().slice(0, 10);
```

**Response shape:**
```typescript
{
  feed: Array<{ id, signalType, companyName, companyDomain, workspaceSlug, title, isHighIntent, detectedAt, status }>,
  typeDistribution: Array<{ name: string, count: number }>,
  summary: { totalSignals: number, totalDailyUsd: number, totalWeeklyUsd: number },
  perWorkspace: Array<{
    slug: string,
    name: string,
    signalsFired: number,
    leadsGenerated: number,
    weeklyUsd: number,
    todayUsd: number,
    dailyCapUsd: number,
  }>,
}
```

**Aggregation logic after queries return:**
- `totalDailyUsd` = sum of costs where `date === todayStr`
- `totalWeeklyUsd` = sum of all costs in the 7-day range
- `perWorkspace` = build from workspace list, merge in costs (weekly + today), signals count (from feed or separate groupBy), leads count (from leadsByWorkspace), and cap from workspace record. For signals per workspace, use `prisma.signalEvent.groupBy` by `workspaceSlug` with the 7-day date filter (add as a 6th parallel query).

Import prisma from `@/lib/db`. Import `NextRequest`, `NextResponse` from `next/server`.
