     ```typescript
     const preWarmActions = await prisma.linkedInAction.findMany({
       where: {
         status: "pending",
         sequenceStepRef: "pre_warm_view",
       },
     });
     ```

  2. **For each pre_warm_view action**, finds the associated connect action:
     ```typescript
     // Same personId + senderId + campaignName, status=pending, actionType is connect or connection_request
     const connectAction = await prisma.linkedInAction.findFirst({
       where: {
         personId: preWarm.personId,
         senderId: preWarm.senderId,
         campaignName: preWarm.campaignName,
         status: "pending",
         actionType: { in: ["connect", "connection_request"] },
       },
     });
     ```

  3. **Checks ordering**: If the pre_warm_view `scheduledFor` is AFTER the connect's `scheduledFor`, it's misordered.

  4. **Fixes misordered actions**: Sets the pre_warm_view's `scheduledFor` to 4 hours before the connect action (or 5 minutes before if the connect is less than 4 hours away):
     ```typescript
     const fourHoursMs = 4 * 60 * 60 * 1000;
     const fiveMinMs = 5 * 60 * 1000;
     const timeBeforeConnect = connectAction.scheduledFor.getTime() - preWarm.scheduledFor.getTime();

     if (timeBeforeConnect <= 0) {
       // Misordered: view is after connect
       const gap = connectAction.scheduledFor.getTime() - Date.now();
       const newScheduledFor = gap > fourHoursMs
         ? new Date(connectAction.scheduledFor.getTime() - fourHoursMs)
         : new Date(connectAction.scheduledFor.getTime() - fiveMinMs);

       if (!dryRun) {
         await prisma.linkedInAction.update({
           where: { id: preWarm.id },
           data: { scheduledFor: newScheduledFor },
         });
       }
       corrections.push({ preWarmId: preWarm.id, connectId: connectAction.id, oldTime: preWarm.scheduledFor, newTime: newScheduledFor });
     }
     ```

  5. **Dry-run mode**: Accept `--dry-run` flag (default behavior). Pass `--apply` to actually make changes.

  6. **Summary report**: Print:
     - Total pending pre_warm_view actions found
     - How many have an associated pending connect action
     - How many are correctly ordered (view before connect)
     - How many are misordered (need fixing)
     - How many were corrected (if --apply)
     - How many had no associated connect (orphaned -- log but don't touch)
